## 完整内容

只有一套键盘鼠标，但需要与多台电脑共用，通常要来回插拔 USB，或蓝牙断开重连，特别麻烦。

刚在 Mac 上写完代码，想切到 Windows 测试，就得把键盘鼠标拔下来重新插到另一台电脑上。

耗时这个肯定的，最重要的还是每次这样来回切换，工作中的思路被打断，严重影响工作效率。

为了解决这个问题，终于在 GitHub 上找到了 Deskflow 这个工具，一套键盘鼠标就直接控制多台电脑，完全开源免费。

不需要额外硬件，只要把鼠标移到屏幕边缘就能"穿越"到另一台电脑上，连剪贴板都能跨设备共享。

跨屏控制，如丝般顺滑

比如前面说的在 Mac 上写代码，需要到旁边的 Windows 电脑上测试程序，可直接把鼠标往右边屏幕滑过去，光标就一下子“飞” 到了 Windows 桌面上，键盘输入也立刻跟了过去。

整个过程没有任何延迟感，就像在用一台超宽屏显示器一样自然。

最让我惊讶的是，即使两台机器运行的是完全不同的操作系统，这种控制感受依然很统一，没有任何割裂感。

剪贴板同步，告别重复操作

另一个让我印象深刻的功能是剪贴板同步。我在 Linux 机器上复制了一段配置代码，直接在 macOS 上就能粘贴使用，完全不需要通过其他工具中转。

这个功能对于经常需要在不同系统间复制文本、链接或者代码片段的人来说简直是神器。

我现在写文档时经常从一台机器上复制资料，然后在另一台机器上编辑，整个流程变得特别自然。

安全加密，默认开启 TLS

在网络安全方面，Deskflow 默认启用了 TLS 加密，这意味着键盘鼠标的控制指令和剪贴板数据在网络传输时都是加密的。

我觉得这个细节很贴心，特别是在企业环境或者公共网络中使用时，不用担心敏感信息被窃取。

其他实用功能

原生 Wayland 支持：对于 Linux 用户来说，Wayland 支持意味着可以在最新的桌面环境中正常使用，不需要回退到 X11；

屏保同步：所有连接的电脑会同时进入和退出屏保状态，保持一致的锁屏体验；

兼容性强：不仅支持 Windows、macOS、Linux，还能与 Synergy、Barrier、Input Leap 等同类软件协同工作；

开源透明：采用 GPL-2.0 许可证，代码完全开放，社区活跃度很高。

安装指南

安装过程不复杂，根据自己的系统选择以下对应方式即可。

1、Windows 系统：

访问 GitHub 发布页面下载最新安装包

双击运行安装程序，按提示完成安装

2、macOS 系统：

brew tap deskflow/homebrew-tap
brew install deskflow

3、Linux 系统：

flatpak install flathub org.deskflow.deskflow
写在最后

对于桌面上有多台设备的朋友来说，Deskflow 确实是个提升工作效率的好工具。

特别是程序员、设计师或者需要在不同系统间频繁切换的用户，这种 “一套设备控制全部” 的体验真的很爽。

不过需要注意的是，所有设备必须在同一个局域网内才能正常工作，而且初次配置时可能需要调整防火墙设置。
