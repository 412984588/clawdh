诞生背景

痛点

• 公共临时邮箱常被滥用、广告轰炸；传统临时邮箱服务（如 10 Minute Mail）依赖中心化服务器，存在隐私泄露风险、访问延迟和域名限制，无法自定义域名或持久化历史记录；
• 自建 Postfix/Dovecot 配置复杂、成本高。

目标

• 构建一个零配置、低成本的临时邮箱系统，100 % Serverless，只依赖 Cloudflare，全球加速；
• 三分钟完成部署，十分钟完成域名级邮箱；
• 同时支持 临时收件 + 域名发件 + 权限隔离。
功能速览
• 现代化界面
• 毛玻璃效果（Glassmorphism）设计，浅色背景与动态渐变，视觉美观；
• 响应式布局，完美适配桌面和移动端；
• 平滑动画和微交互反馈，提升用户体验。
• 邮箱管理
• 随机生成临时邮箱地址，支持自定义长度和域名（多域名逗号分隔）；
• 自动保存历史记录，便于重复使用；
• 支持删除单个邮箱或批量管理；
• 一键切换邮箱，置顶功能（V2 起）。

安装 部署
1. 二进制/部署推荐
• 使用 Wrangler CLI：安装 npm i -g wrangler，登录 wrangler login；
• 创建 D1：wrangler d1 create temp-mail-db，绑定 wrangler.toml；
• 设置变量：wrangler secret put ADMIN_PASSWORD 等（Secrets），[vars] 配置 MAIL_DOMAIN 等；
• 本地开发：wrangler dev；
• 线上部署：wrangler deploy，或 Cloudflare Dashboard 连接 GitHub 仓库（Variables 配置变量）；
• 初始化 D1：wrangler d1 execute TEMP_MAIL_DB --file=./d1-init-basic.sql（若未自动创建）。
• 邮件路由：Cloudflare Email Routing 添加 Catch-all 规则，目标 Worker。
• 自定义域名：Worker 设置中添加，或用 workers.dev。
2. 源码构建
• 克隆 git clone https://github.com/idinging/freemail.git；
• 编辑 wrangler.toml 和变量；
• 运行 wrangler dev/deploy。
3. 更新
• Git pull 拉取最新；重新 deploy；静态缓存需 Purge Everything。
使用流程
1. 部署后访问 Worker URL 或自定义域名；
2. 登录：输入用户名（默认 admin）和密码（ADMIN_PASSWORD），获取 JWT；
3. 生成邮箱：点击随机生成，支持自定义；历史记录自动保存；
4. 接收邮件：选中邮箱，每 8 秒刷新，查看列表/详情，复制验证码；
5. 管理用户：严格管理员进入“用户管理”，创建/编辑用户，分配邮箱；
6. 发件：配置 RESEND_API_KEY 后，输入内容发送，支持批量/定时；查看发件箱；
7. 删除/清空：单个或批量操作邮件/邮箱。
典型场景
• 临时注册：生成邮箱接收验证码，避免隐私泄露；
• 域名营销：批量发信 + 定时取消，配合 Resend 统计 ；
• 团队协作：管理员创建用户，分配权限/邮箱；
• 内容分享：发件推广，查看记录。
