🛠️ 想拥有Suna？你需要这些“装备”！

想让Suna为你服务？没问题！不过，这位AI大牛也有点“小讲究”，你需要为它准备好以下“豪华套餐”：

数据库 & 认证管家 (Supabase)：
需要一个Supabase项目来存储数据和管理用户。
缓存 & 会话好帮手 (Redis)：
让数据飞起来~
安全执行环境 (Daytona)：
给Suna一个安全的小黑屋去执行任务。
编程语言 (Python 3.11)：
Suna的后端是用Python写的。
大模型通行证 (LLM API Keys)：
需要OpenAI或Anthropic的API密钥（当然，通过LiteLLM理论上也支持其他的）。
(可选) 超强搜索外挂 (Tavily API Key)：
想让搜索能力更上一层楼？搞个Tavily的密钥吧！
(可选) 更多技能解锁 (RapidAPI Key)：
想用领英等更多第三方服务？需要RapidAPI的密钥并订阅相应服务。

 如何召唤Suna？保姆级教程来了！

准备好亲自体验Suna的强大了吗？跟着下面的步骤，你就可以在本地把它运行起来：

克隆代码仓库：

git clone https://github.com/kortix-ai/suna.git
cd suna
2. 配置后端环境：
cd backend
cp .env.example .env # 如果没有.env.example，手动创建一个.env文件

然后，编辑 .env 文件，填入你之前准备好的各种凭证（Supabase, Redis, Daytona, LLM Keys等）。具体需要填写的可以参考项目 README.md 的详细说明。

3. 设置Supabase数据库：

# 登录Supabase CLI
supabase login
# 关联你的项目
supabase link --project-ref your_project_reference_id
# 应用数据库变更
supabase db push

之后，还需要去Supabase网页后台做一些配置，具体参见 README.md。

4. 配置前端环境：

cd ../frontend
cp .env.example .env.local # 同理，创建 .env.local 文件

编辑 .env.local 文件，填入Supabase的URL、Anon Key以及后端API地址。

5. 安装依赖：

# 安装前端依赖
cd frontend
npm install


# 安装后端依赖
cd ../backend
pip install -r requirements.txt

6. 启动应用：

打开一个终端，启动前端：
cd frontend
npm run dev
再打开一个终端，启动后端：
cd backend
python api.py
开始使用： 
在浏览器中打开 http://localhost:3000，注册一个账号，然后就可以开始和你的专属Suna智能体对话啦！🎉
