     mitmproxy 和此前介绍的 ACME 工具均为开源软件，遵循免费使用的原则，其源代码均托管在 GitHub 平台。它是一个交互式的、可用于拦截、检查、修改和重放网络流量的中间人代理工具。所有经过它的网络请求和响应，你都能看得一清二楚，甚至可以动手修改。如此强大的工具接下来让我们一起来了解它

     此时，HTTP 请求已可正常转发至 mitmproxy，在终端里能清晰看到请求与响应的所有报文细节。举个常见的实际用法：如果你在电脑端开发本地项目（访问地址为 http://xxx.local），想在手机端验证效果，只需满足两个条件：一是手机与电脑处于同一网络，二是手动给手机配置代理 ——IP 填电脑的局域网 IP，端口填 mitmproxy 的代理端口 8080。假设电脑的局域网 IP 为 192.168.0.5，手机端的具体设置如下图所示：

     手机首次连接代理服务器时，可能会有约两三分钟的网络延迟，导致暂时无法上网。此时只需在电脑端重新执行 mitmproxy 命令重启代理，稍作等待网络即可恢复正常。之后，在手机浏览器中输入 http://xxx.local 即可访问电脑上的项目。

     若需处理 HTTPS 请求，仅靠上述基础配置还不够，必须额外在手机和电脑端安装 mitmproxy 专属的根证书，之后 HTTPS 相关的网络连接才能顺利生效。具体安装步骤如下：首先确保手机已成功连接代理，接着打开手机自带浏览器，输入并访问 mitm.it；选择手机系统匹配对应的证书，点击下载后，跟随系统弹出的安装指引逐步操作，就能完成 mitmproxy 手机证书的安装。

对于mitmproxy 工具集包含了三个核心终端命令，他们各有侧重：

1、mitmproxy：以纯命令行交互形式呈现抓包信息，无需图形界面，适合在服务器环境或偏好高效命令操作的场景使用；

2、mitmweb：通过 Web 界面展示抓包数据，启动后可在浏览器中操作，支持可视化筛选与分析，易用性更强；

3、mitmdump：主打抓包数据的脚本化处理，能将流量转发至 Python 脚本，方便开发者编写自动化逻辑。从技术架构来看，该开源项目的前端基于 HTML、JS、CSS 构建，后端则通过 Python 实现核心功能。
