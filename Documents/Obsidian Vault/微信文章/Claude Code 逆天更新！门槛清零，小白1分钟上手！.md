一、Claude Code 又更新了！

这两天我重新关注了一下 Claude Code（简称 cc），发现它又有大动作。 最大变化是——安装方式彻底简化了！

以前安装 cc，要折腾各种环境、包、版本……

现在呢？只要一行命令，啥都不用装，谁都能用。

我看到这更新的第一反应就是： “普通人的门槛，彻底清零了！” AI 编程的入场券，终于发到每个人手里了。

二、以前有多麻烦？

老用户应该都懂。

当年要装 cc，我们得先来一套“仪式”：

npm install -g @anthropic-ai/claude-code

听起来简单？其实不然。 因为这背后有个隐藏条件——你必须先安装 Node.js 和 npm 。 对技术人来说，这叫“配置环境”； 对普通人来说，这叫“劝退现场”。

更麻烦的是：

Node 版本不同，可能会冲突；

macOS 和 Windows 的安装方式还不一样；

有时候还得改路径、装依赖、升级包管理器……

很多想尝鲜的用户，刚兴冲冲打开教程，结果看到一堆命令行就关了。

毕竟不是谁都喜欢看黑框框的窗口嘛。

我一直觉得，一个好产品，应该是 “上到80岁，下到三岁”都能用的。

显然，之前的 cc 还没做到这一点。

三、这次更新，有点意思！

但这次，cc 的产品经理显然开了窍。

他们直接把那些繁琐的依赖环境，都打包进了原生安装包里。

换句话说， 你再也不用去装 Node.js、npm 这些乱七八糟的东西了。 只要复制一行命令，粘贴回车，就能用。

来看下新版安装方式👇

✅ macOS / Linux / WSL 用户：
curl -fsSL https://claude.ai/install.sh | bash

或者用 Homebrew：

brew install --cask claude-code
✅ Windows 用户：
irm https://claude.ai/install.ps1 | iex

全程不超过 10 秒。

再也没有“环境冲突”“权限报错”这些头疼事儿了。

安装完之后，打开终端输入：

claude

就能直接启动。 真正的一行命令，世界和平。

四、老用户怎么办？

有朋友要问了：“我之前是按老方法装的，那还能切换成新方式吗？”

当然可以！步骤也超简单👇

1️⃣ 先关闭所有 Claude Code 会话窗口，包括终端和 VSCode 的。

2️⃣ 然后在终端依次输入以下命令：

claude install
npm uninstall -g @anthropic-ai/claude-code

前一句是装新版本，后一句是卸载旧的。

整个过程也就喝一口水的时间。
