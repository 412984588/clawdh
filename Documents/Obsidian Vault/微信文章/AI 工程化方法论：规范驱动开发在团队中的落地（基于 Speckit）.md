# AI 工程化方法论：规范驱动开发在团队中的落地（基于 Speckit）

背景

过去大半年，我把更多“编码”交给了 AI，自己专注于目标定义、上下文提供与节奏把控。与 AI 协作的过程中，我更像是一名“AI 指挥家”，并逐步形成了稳定的协作流程：

请明确项目需求及预期实现的效果。
请调用AI处理文档#1，确认是否理解了所提出的需求，并基于此制定初步方案。如果在理解和方案设计过程中遇到任何问题，请与我进行讨论。
根据AI反馈的问题给出解答，并进一步细化具体的行动计划和任务清单。
开始编码工作，在完成每个重要里程碑后，请及时向我报告进度并寻求确认。

按这套流程推进后，大多数功能能一次成型；收尾阶段由 AI 生成单元测试，我做验收。 痛点：每个功能点往往伴随大量确认项，沟通容易机械、重复。 接触 Speckit 之后，我发现它把上述协作流程标准化、可复制。

是什么

它定义并实现了一种名为“规范驱动开发”（Spec-Driven Development, SDD）的软件开发方法论，旨在通过与 AI 编码助手（如 GitHub Copilot, Claude, Gemini 等）协作，更加结构化、可预测地构建高质量软件。

如何理解？

:::warning 其通过内置各个AI编码助手的自定义命令的方式，将开发过程中的流程进行规范化，并充分按照生成文档进行。

:::

先了解其提供的一系列核心命令：

**初始化项目 (specify init): **开发者首先使用仓库提供的 Python 命令行工具 specify-cli 来初始化一个项目。这个命令会根据你选择的 AI 助手（如 copilot），在你的项目中创建一套标准的目录结构、模板文件和自动化脚本。
/speckit.constitution：建立项目原则，告诉 AI 这个项目的“宪法”是什么，比如编码规范、测试标准、技术选型原则等。 AI 会将这些原则记录在 memory/constitution.md 文件中，作为后续所有决策的最高准则。
/speckit.specify：创建功能规范，用自然语言描述“你想构建什么功能”（What & Why），比如：“我想做一个照片管理应用，可以创建相册...”。AI 会调用 create-new-feature.sh 脚本，自动创建一个 Git 分支（如 001-photo-albums），并根据 spec-template.md 模板，生成一份详细的功能规范文档 spec.md。这份文档包含了用户故事、功能需求、验收标准等。
/speckit.plan：创建技术实现计划，告诉 AI “你想用什么技术来实现”（How），比如：“用 Vite + 原生 JS，数据存在本地 SQLite”。AI 会根据功能规范 (spec.md) 和你的技术要求，生成一份详细的技术计划 plan.md，可能还包括数据模型、API 合同等。
/speckit.tasks： 分解任务，AI 会分析技术计划，并将其分解成一个详细的、可执行的任务列表 tasks.md，其中包含了每个任务需要创建或修改的具体文件路径。
/speckit.implement： 执行实现，AI 会按照 tasks.md 中的任务列表，一步一步地编写代码、创建文件、运行命令，最终完成整个功能的开发。

为什么这么理解呢？

答：spec-kit提供了整个开发生命周期，每个生命周期会通过不同的内置的prompt+宪法，让ai对于需求，技术方案更加了解，最后在编码阶段进行更加准确的生成。

说人话，流程+沟通较差的程序员，也能在这个模式下生成比较符合规范的代码。

每个命令执行后，都会进行对应的阶段，在阶段内的对话都是将这一步做好。 有没有发现，这个流程其实跟我们做事方式很像？一个标准的实现路线，将每一步做好，达成最后的成功。

落地建议

工程规范先行

AI 辅助的前提是“工程可被机器理解”。如果项目结构与规范缺失，AI 的增益会明显受限；反之，收益会成倍放大。

Speckit 每一次功能的开发都会依赖于《宪法》。可通过 /speckit.constitution 在项目级进行维护与更新。

Go 全栈实践要点

对于新老项目代码，都可以采取以下几个准则：

项目结构准则
日志规范
配置管理规范
API 规范：请求体、响应体一致性
前端开发规范
Go 最佳实践
TDD 测试驱动开发（非协商）

具体规范可在调用 /speckit.constitution 时与 AI 讨论，并在迭代中持续完善，最终显著提升团队效率。

总结

AI辅助编码技术的发展速度远超预期，其能力早已超越了简单的代码生成范畴。例如，我针对集团内部告警平台的重构工作，基于一个开源项目进行了二次开发，并实现了与内部权限管理系统的集成、项目规范的整理、遵循公司部署标准、迁移原有平台规则以及开发用于展示迁移进度的可视化界面等功能。从整体架构梳理到最终部署测试，整个过程仅耗时一周左右，这期间还包括了对产品设计思路的深入探讨。

 此外，就知识获取及准确性而言，当前AI的表现与半年前相比已不可同日而语。随着这项技术不断进步，掌握AI技能与否将逐渐成为区分程序员能力水平的重要标志之一，两者之间的差距也将日益明显。 因此，持续学习并紧跟最新技术趋势对于每一位IT从业者来说都至关重要。
