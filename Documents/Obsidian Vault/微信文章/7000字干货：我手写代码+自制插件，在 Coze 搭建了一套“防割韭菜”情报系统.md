在这个“项目满天飞，镰刀遍地走”的时代，信息差已经不是最值钱的了，**“鉴别力”**才是。

很多朋友问我：“你的那个能自动找项目、还能自动查风险的智能体是怎么搭的？”

传统的问答机器人只能陪聊，而我要做的是一个**“全自动化的商业情报流水线”**。

今天，我决定开源这个工作流的构建全过程。我会结合后台真实配置截图，把每一个节点的设置、代码逻辑、Prompt 技巧全部掰开了揉碎了讲清楚。

建议先收藏，打开 Coze.cn（扣子）官网，跟着我一步步复现。

01. 全局设计：正反双向搜索机制



首先，看一眼上帝视角（全景图）：

这个工作流的核心逻辑在于**“正反合一”**：

正向挖掘： 全网搜索搞钱机会。

中间转化： 提取关键信息，并生成“排雷关键词”。

反向侦察： 拿着“排雷词”去全网搜差评、搜骗局。

动态决策： 根据正反两面的信息，决定是给“SOP”还是给“劝退报告”。




下面我们逐一拆解十大核心节点。

02. 第一阶段：情报清洗（Input & Cleaning）

用户输入的往往是口语（如“想找个副业”），直接拿去搜 Google 效果很差。前两个节点负责把“口语”变成“机器语言”。

节点 1：情报搜集员 (LLM)

模型： 豆包·1.5·Pro·32k

角色设定： 精通 Google Hacking 的“商业情报猎手”。

任务： 它的嗅觉敏锐，不直接回答问题，而是将用户的模糊需求，翻译成适合搜索引擎的高级检索语法。

# Role
你是一位精通 Google Hacking 的“商业情报猎手”。你的嗅觉敏锐，擅长通过组合关键词，挖掘互联网上隐藏的蓝海商机、暴利项目、实操教程和变现案例。


# Task
根据用户输入的【主题】，构造一个**高价值的正向搜索指令 (Search Query)**。


# Strategy
不要搜索泛泛的概念，要搜索具体的“实操痕迹”。请灵活使用以下搜索逻辑的组合：
1. **找教程/SOP**: 关键词包括 "教程", "实操", "保姆级", "SOP", "玩法", "攻略"。
2. **找数据/收益**: 关键词包括 "变现", "收益", "晒单", "日入", "复盘"。
3. **限定高价值平台**: 结合 `site:xiaohongshu.com` 或 `site:zhihu.com` 或 `site:tiktok.com` (根据主题自动判断)。
4. **排除干扰**: 使用 `-` 排除 "课程", "培训" (我们要找免费分享，不要找卖课广告)。


# Input Analysis
用户输入: {{user_input}}


# Output Format
请直接输出一个 JSON 对象，包含唯一的搜索词字段。不要输出任何其他解释。


示例格式:
{
  "search_query": "site:xiaohongshu.com 'AI绘图' (变现 OR 收益 OR 接单) -卖课"
}

节点 2：搜索词清洗器 (Code)

类型： Python 代码节点

作用： 这是一个保险丝。有时大模型输出会带废话，这个 Python 脚本负责利用正则提取纯净的 Search Query，确保传给 Google 的参数是干净的。

import json
import re


async def main(args):
    params = args.get('params', args)
    # 1. 获取输入 (LLM 1 的输出)
    raw_text = params.get("raw_text", "")


    # 默认兜底词
    final_query = str(raw_text)[:50] 


    try:
        # 2. 清洗 JSON
        clean_str = str(raw_text).strip()
        if "```" in clean_str:
            clean_str = re.sub(r'^```json\s*|^```\s*|```$', '', clean_str, flags=re.MULTILINE).strip()


        # 3. 解析并提取
        data = json.loads(clean_str)
        if isinstance(data, dict):
            # 直接提取 LLM 精心构造好的搜索词
            final_query = data.get("search_query", final_query)


    except Exception:
        # 如果解析失败，说明 LLM 可能直接输出了字符串，直接用清洗后的字符串
        final_query = clean_str


    # 4. 返回给 Search 1
    return {
        "search_query": final_query
    }
03. 第二阶段：正向挖掘 (Discovery)

这是工作流的“眼睛”。注意，这里我没有用 Coze 自带的搜索，而是用了自制的 Google Search 插件。(平台第三方的Google Search插件不好用，自己做了一个，我会出一个教程，教怎么制作）

节点 3：正向搜索 (Custom Plugin)

配置： 输入 cx (搜索引擎ID) 和 key (API Key)，以及清洗后的 q (关键词)。

为什么要自制？ 因为 Google 的搜索质量在商业情报领域依然是天花板，特别是针对海外项目（TikTok、跨境电商）的挖掘。

节点 4：商业分析师 (LLM)

模型：豆包·1.6·自动深度思考（注意：这里用了更强的推理模型）

核心功能： 它是整个流程的第一个“大脑”。它不仅要总结项目，更要埋下探针。

Prompt 必杀技： 我强制要求它输出 JSON 格式，并且必须包含一个字段叫 risk_search_query（风险搜索词）。

比如搜到“无人直播”，它必须自动生成：“无人直播 + (骗局 OR 封号 OR 割韭菜)”。

# Role
你是一位风险厌恶型的顶级商业分析师。你刚刚收到了一份关于某项目的【正向搜索结果】。


# Task
1. **提炼方案**: 从搜索结果中归纳出一个最具体的变现项目（SOP）。
2. **多维评分**: 评估它的操作难度、变现潜力和风险等级。
3. **构造反向验证**: 基于你归纳的项目，构造一个用于“打假”的搜索词。


# Output Format
请严格输出 JSON 格式（不要 Markdown）：
{
  "schemes": [
    {
      "name": "项目名称 (例如：TikTok无人直播带货)",
      "description": "详细的实操逻辑描述...",
      "risk_level": "高风险", // 预估风险
      "difficulty": 3,
      "potential": "S级(暴利)",
      "advice": "一句话执行建议",
      "risk_search_query": "项目名称 + (骗局 OR 封号 OR 割韭菜 OR 避坑)" // 👈 这一步非常重要！
    }
  ]
}
04. 第三阶段：中枢分发 (The Hub)

这是区分新手和高手的关键一步。

节点 5：情报分发中枢 (Code)

类型： Python 代码节点

作用： 解析上一步的 JSON 数据，进行流量分发。

把 risk_search_query 提取出来，扔给“照妖镜”去查风险。

把 scheme_info 提取出来，扔给“执行总监”做 SOP。

把结构化数据提取出来，准备存入数据库。

代码逻辑： 利用 json.loads 解析大模型返回的字符串，精准分配给下游。




import json
import re
import time


async def main(args):
    # --- 1. 获取输入 ---
    params = args.get('params', args)
    # 兼容 LLM 的输出变量名，优先读 raw_text，读不到读 json_input
    raw_input = params.get("raw_text") or params.get("json_input")


    data = {}


    # --- 2. 强力解析 JSON (核心逻辑) ---
    try:
        if raw_input is None:
            raise ValueError("输入为空")


        if isinstance(raw_input, dict):
            # 如果 Coze 已经自动转成了对象
            data = raw_input
        else:
            # 如果是字符串，先清洗 Markdown 符号
            clean_str = str(raw_input).strip()
            if "```" in clean_str:
                clean_str = re.sub(r'^```json\s*|^```\s*|```$', '', clean_str, flags=re.MULTILINE).strip()
            data = json.loads(clean_str)     
    except Exception as e:
        # 兜底：解析失败时返回默认值，防止工作流报错停止
        return {
            "search_query": "TikTok 风险", 
            "records_out": [],
            "info_for_final": "{}",
            "summary_for_chat": f"❌ 数据解析失败，请检查 LLM 输出格式。\nError: {str(e)}"
        }


    # --- 3. 提取核心数据 ---
    schemes = data.get("schemes", [])
    if not schemes:
        return { 
            "summary_for_chat": "⚠️ 未提取到有效方案，请检查上一步分析结果。", 
            "records_out": [],
            "search_query": "无数据",
            "info_for_final": "{}"
        }


    item = schemes[0] # 取第一个方案


    # 提取字段 (给变量赋默认值，防止缺字段报错)
    name = item.get("name", "未命名项目")
    risk = item.get("risk_level", "中风险")
    poten = item.get("potential", "B级")
    advice = item.get("advice", "暂无建议")
    diff = item.get("difficulty", 3)
    desc = item.get("description", "无描述")


    # 提取反向搜索词
    risk_query = item.get("risk_search_query")
    if not risk_query:
        risk_query = f"{name} 骗局 风险"


    # --- 4. 构造输出数据 ---


    # A. 给飞书多维表格 (Key必须和飞书列名一致)
    fields_dict = {
        "项目名称": name,
        "项目描述": desc,
        "风险等级": risk,
        "操作难度": int(diff),
        "变现潜力": poten,
        "执行建议": advice,
        "记录时间": int(time.time() * 1000)
    }
    # 飞书插件要求：records 必须是数组，fields 必须是字符串
    records_array = [{"fields": json.dumps(fields_dict, ensure_ascii=False)}]


    # B. 给群聊界面 (End节点) 的漂亮摘要
    summary_text = (
        f"📊 **项目评级报告**\n"
        f"━━━━━━━━━━━━━━\n"
        f"🏷️ **项目名称**: {name}\n"
        f"⚡ **变现潜力**: {poten}\n"
        f"🛡️ **风险等级**: {risk}\n"
        f"💡 **专家建议**: {advice}\n"
        f"━━━━━━━━━━━━━━"
    )


    # --- 5. 返回结果 (对应左侧 Output) ---
    return {
        "search_query": risk_query,   
        "records_out": records_array, 
        "info_for_final": json.dumps(item, ensure_ascii=False),
        "summary_for_chat": summary_text 
    }
05. 第四阶段：反向排雷 (Risk Control)

这是本工作流最性感的部分——“照妖镜”。

节点 6：风控官（照妖镜） (Custom Plugin)

原理： 再次调用 Google Search 插件。

输入： 这里的 q 不再是“怎么赚钱”，而是代码节点分发过来的**“负面关键词”**（如：xx项目 骗局）。

输出： 它会搜出论坛吐槽、曝光贴、避坑指南。这才是真实的互联网声音。




06. 第五阶段：决策与执行 (Decision & Output)

到了这一步，AI 手里握着“项目介绍”和“风险情报”两份档案，开始做最终裁决。

节点 7：执行总监 (LLM)

模型： 豆包·1.5·Pro·32k

输入：scheme_info (赚钱逻辑) + risk_info (风险情报)。

Prompt 策略： 我设置了动态模式（Mode A/B）。

如果 risk_info 里全是骂声 -> 触发模式 A：输出《风险劝退报告》，严厉警告！

如果风险可控 -> 触发模式 B：输出《项目实操 SOP》，并将搜到的坑点写入“避坑指南”。




# Role
你是一位拥有 10 年经验的**互联网项目操盘手**，擅长制定标准作业程序 (SOP) 和风险控制。你的行事风格稳健、务实，从不盲目画饼。


# Task
你收到了一个【项目方案】和一份对应的【风险尽职调查报告（搜索结果）】。请综合这两份信息，为用户输出一份**《项目落地执行 SOP》**或**《风险劝退报告》**。


# Inputs
1. **项目方案**: {{scheme_info}}
2. **风险情报**: {{risk_info}}


# Critical Logic (核心判断逻辑)
首先，请仔细阅读【风险情报】。
- **情况 A (高风险)**: 如果搜索结果中充斥着“骗局”、“割韭菜”、“封号严重”、“无法提现”、“跑路”等负面信息 -> **请立即启动劝退模式**，不要输出 SOP，而是输出风险警示。
- **情况 B (可行)**: 如果风险可控，或者负面信息多为操作不当导致 -> **请输出详细的实操 SOP**，并在 SOP 中加入专门的“风控环节”。


# Output Format (根据判断结果选择一种)


## 模式 A：风险劝退报告 (当风险极高时)
### 🛑 严重风险预警
- **项目定性**: (例如：资金盘 / 杀猪盘 / 纯割韭菜项目)
- **核心风险点**:
    1. ...
    2. ...
- **专家结论**: 强烈建议放弃！不要浪费时间。


---


## 模式 B：项目实操 SOP (当项目可行时)
### 🚀 项目启动书: {项目名称}


#### 1. 商业逻辑拆解
- **搞钱原理**: (一句话讲清楚赚谁的钱，怎么赚)
- **核心卖点**: ...


#### 2. 风险规避指南 (基于风险情报)
*(针对搜到的坑，给出解决方案)*
- **潜在坑点**: ...
- **应对策略**: ...


#### 3. 保姆级执行步骤 (Step-by-Step)
- **第一阶段：准备期 (Day 1-2)**
    - 账号准备: ...
    - 工具配置: ...
- **第二阶段：执行期 (Day 3-7)**
    - 内容制作: ...
    - 流量获取: ...
- **第三阶段：变现期**
    - 转化话术: ...
    - 交付流程: ...


#### 4. 必备工具箱
- (列出需要的 AI 工具、软件、网站)


# Constraints
- 语气要客观、专业、干练。
- 如果搜索结果为空，默认按“中低风险”处理，但在 SOP 中提示“需小规模测试”。
- 输出格式使用 Markdown，标题层级清晰。

节点 8：公众号文章 (LLM)

作用： 同时并行，将这次调研整理成一篇风格犀利的公众号文章草稿，方便我直接发布。

人设： 设定为“深耕 AI 与副业的主理人”，文风犀利、客观。




# Role
你是一位深耕 AI 与副业领域的**公众号主理人**（类似“狂飙”、“阿禅”的风格）。你的文风犀利、客观，擅长用大白话拆解复杂项目，既能通过“揭秘”带来爽感，又能提供扎实的“干货”。


# Task
根据提供的【原始项目档案】和【核心分析报告（SOP/劝退书）】，撰写一篇公众号文章。


# Inputs
1. **项目档案**: {{project_info}}
2. **核心报告**: {{sop_core_content}}


# Article Strategy (自动判断文章基调)
请先阅读【核心报告】的内容：


**场景 A：如果报告是“风险劝退 / 严重预警”**
- **标题风格**: 警示类。例如《紧急避坑！{项目名}是新型杀猪盘？》、《别再被割了！深扒{项目名}的底层逻辑》。
- **文章情绪**: 愤怒、揭秘、冷静分析。
- **写作重点**: 拆解骗局套路 -> 展示证据 -> 告诉读者如何防范。


**场景 B：如果报告是“项目实操 SOP”**
- **标题风格**: 红利类。例如《信息差！{项目名}的保姆级玩法》、《0 成本！我用 AI 在{项目名}搞到了第一桶金》。
- **文章情绪**: 兴奋、发现新大陆、鼓励尝试。
- **写作重点**: 抛出收益/商机 -> 拆解具体步骤 (引用 SOP) -> 强调风控 -> 鼓励执行。


# Writing Structure (通用结构)
1. **标题 (Title)**: 给出 3 个爆款标题供选择。
2. **开头 (Hook)**:
   - 从痛点切入（"最近很多人在后台问我..." 或 "昨天我在小红书刷到一个..."）。
   - 制造悬念，引导阅读。
3. **深扒/拆解 (Body)**:
   - 介绍这到底是个什么项目（利用项目档案）。
   - **(核心)** 直接引用并润色【核心报告】中的干货内容。如果是 SOP，就列出步骤；如果是骗局，就列出疑点。
   - 使用加粗、Emoji (💡, 🚀, 🛑) 优化排版，避免大段文字。
4. **结论与建议 (Verdict)**:
   - 给出最终的一句话狠评（做，还是不做？）。
   - "风险等级：⭐⭐⭐"（根据数据可视化展示）。
5. **结尾 (Call to Action)**:
   - 引导点赞/在看（"觉得有用的，文末点个在看"）。
   - 引导关注（"关注我，带你拆解更多 AI 变现玩法"）。


# Constraints
- **排版**: 使用 Markdown 格式。小标题加粗。
- **口语化**: 像在和朋友聊天，不要用翻译腔。
- **长度**: 1500 字左右。

💡 关键知识点：如何获取 Token 和 ID？打开你的飞书多维表链接，看浏览器地址栏：https://ai.feishu.cn/base/appxxxxxxxxxx?table=tblxxxxxxxxxx

app 后面的 app... 这一串是 App Token（这里支持长链接，这个链接扔过去也可以）。

table= 后面的 tbl... 这一串是 Table ID。 这两串字符，一会儿要填进 Coze 的插件配置里（选填）。

回到 Coze，添加 Feishu Multi-BitTable 插件的 add_record 工具。

app_token / table_id： 把刚才飞书链接里提取的字符串填进去。

records (核心)： 这里千万不能乱填！点击引用变量，选择上一步 “情报分发中枢” 代码节点输出的 records_out 变量。




第三步：代码节点的“暗号”对接（核心解密）

为什么我在 Coze 里选一个 records_out 就能自动填表？因为我在 Python 代码里已经做好了格式封装。

大家回看上一节的代码，注意 fields_dict 这个字典：

# A. 构造飞书多维表格数据 (Key必须和飞书列名完全一致)
    fields_dict = {
        "项目名称": name,
        "项目描述": desc,
        "风险等级": risk,
        "操作难度": int(diff),
        "变现潜力": poten,
        "执行建议": advice,
        "记录时间": int(time.time() * 1000)
    }
    # ⚠️ 知识点：飞书插件要求 records 是数组，且 fields 内部必须是 JSON 字符串
    records_array = [{"fields": json.dumps(fields_dict, ensure_ascii=False)}]

这里的逻辑非常严密：

Key 的一一对应： 代码里的 "项目名称" 必须等于飞书表头的 “项目名称”。

JSON 序列化： 飞书 API 比较特殊，它要求 fields 的值必须是一个 String（字符串），而不是 Object（对象）。所以我在代码里用了 json.dumps 帮你转好了。

实战效果：配置完成后，智能体每跑一次，表格里就会自动增加一行。 你看，这是我跑了一下午的成果，**“风险等级”**那一列一目了然，那些标着“高风险”的项目，我连看都不用看，直接 Pass。

这就是自动化的力量——让数据为你打工，而不是你为数据录入打工。
