Claude Code实现Manus般的规划实践体验

介绍一个开源项目：https://github.com/OthmanAdi/planning-with-files
3天，涨了1.2K的star，目测还在继续升温。

这个项目给我最大的启发就是，用最简单易懂的方式，组合了最新最热的技术，实现了拟人的效果。

大致意思就是，ClaudeCode有一些固有的缺陷，比如脆弱的记忆，目标的漂移，错误被隐藏，上下文中填充了全部内容，导致处理复杂任务的时候比较吃力。

所以，作者就参考了manus的方式，通过记笔记的方式来管理上下文。就像一个小朋友，先把老师们布置的所有的作业记录在本本上，语文作业，数学作业，英语作业等，满满一页。然后，做完一个，勾掉一个，直到全部做完，结束。

好记性不如烂笔头，在这里得到了完美的诠释。对于小朋友，对于AI都是这样的。

这个系统，就是用Claude Code Skill的技术，让AI在规划的时候，把需要完成的“作业”都写在文件里，然后每次执行具体的操作前，先读下规划，执行完后，再更新下规划。这样就让AI每次都知道自己当下具体要做的是什么，同时又不会忘记最终的大的目标是什么。项目进度，清清楚楚。

这个项目可以称得上是Tiny版的Manus，期望作者能补充下多任务并行执行，和多笔记保存的功能。毕竟，对于高年级的小朋友，2张纸的笔记可能不够了
