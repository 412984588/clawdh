# Claude Code + Cursor + MCP：打造10x开发者工作流

> 作者: AI容人懒 | 发布日期: 2025年12月5日 08:31
> 原文链接: [点击查看](https://mp.weixin.qq.com/s/Ipqrvjfp7L7sFSGLy8oU1g)

---

昨天看 Greg 对谈独立开发者 Chris 的播客时,我注意到一个细节。




屏幕左边是 Cursor,右边的终端里跑着 Claude Code。Chris 在两个 AI 之间来回切换,像指挥两个助手同时干活。

这个场景让我愣住了。因为过去三个月,我一直在纠结该用哪个工具——要么 Claude Code,要么 Cursor,必须二选一。

但 Chris 的月收入已经过万美金。他用 AI 做了好几款产品应用,全是一个人搞定。




他的答案不是"选哪个更好",而是"为什么要选"?

当所有人在找最佳工具时,高手在组装工具

他说得很直白:“Claude Code 的 Opus 4.5 模型,可能是现在最强的编码模型,但限额太少,我三四个小时就能用完。所以我只在最复杂的问题上用它。”




“Cursor 的 Plan Mode 配合 GPT-5.1-high,特别擅长规划和追问。遇到复杂 bug 时,我会先让它做计划,再执行。”




他展示了一个动画效果的对比测试。同样的需求,Claude Code 第一次输出的效果一般。但 Cursor 用 Plan Mode 第一次就做得更好——流畅的下落动画,精确的时间控制,还有那个闪烁的高光效果。

区别在哪?Plan Mode 让 AI 先花 30 秒思考,帮你省下 30 分钟返工时间。




你写周报时,是不是直接让 ChatGPT 一口气生成,然后花半小时改得面目全非?

换个思路:让它先列提纲,你确认思路没问题,再让它展开。快了吗?快多了。

Chris 说这个方法至少提升 20% 输出质量。




但更重要的是,他根本不纠结"哪个工具更好"。他知道每个工具擅长什么,然后像搭积木一样组合使用。

UI 设计? Claude 更敏锐。

复杂 bug? Cursor Plan Mode 更靠谱。

超级复杂的架构?Opus 4.5 上场。




这就是"工具栈思维"。

不是找完美工具,而是构建工具组合。就像厨师不会只用一把刀,程序员为什么要只用一个 AI?

那个让 Claude 更努力工作的关键词

Chris 提到一个细节时,我把播客倒回去听了2遍。

“有个特殊关键词叫 ultrathink。你在 Claude Code 里输入它,字会变色,因为它触发了隐藏指令——让 Claude 更深度地思考问题。”

“我在 90% 的提示词里都会用它。”




我立刻试了一下。确实,输入 ultrathink 后,Claude 的响应时间变长了,但给出的方案明显更周全。

这个发现让我意识到一件事:我们用工具,但从不研究工具本身。

就像你天天开车,但从不看说明书里那些隐藏功能。别人知道方向盘上的小按钮能激活自适应巡航,你还在傻傻地踩油门。

这就是"隐藏杠杆"。

系统里总有未公开的高级功能,发现它们的人获得不对称优势。

但发现它们需要什么?需要你停止焦虑"用哪个工具",开始研究"这个工具为什么这样设计"。




Chris 还分享了其他隐藏杠杆:

后台任务:让 Claude Code 在后台运行服务器,它就能实时看到报错日志,调试效率翻倍。

MCP 服务器:接入 Context 7 后,AI 能直接调用最新文档,不用你复制粘贴。接入 Supabase MCP,它甚至能帮你配置数据库安全规则——Chris 说它配得比他自己手动配还安全。

代码审查工具:他用 Bugbot 自动审查每次提交,抓到了无数他没注意到的安全漏洞。每月 40 美元,但"让我能睡得着觉"。

这些都不是秘密,但 99% 的人不知道。

为什么?

因为大家都在问"哪个工具好",没人问"这个工具还能怎么用"。

你是在选工具,还是在逃避思考?

说到这里,我想问你一个不太舒服的问题。

当你纠结"该用 Claude 还是 GPT、该用 Cursor 还是 Windsurf"时,你真的是在做技术决策吗?

还是说,你在寻找一个"正确答案",好让自己不用再思考?




Chris 能月入过万,不是因为他选对了工具。而是因为他理解每个工具的边界、优势和局限,然后构建了自己的组合系统。

这套系统不是固定的。他说:“几个月后肯定会换。新模型会出来,新工具会变强,我会不断调整。”

真正的效率,不是找到一劳永逸的方案, 而是保持持续判断的能力。




这很累。

因为这意味着你不能停止学习。你必须试错、比较、调整。你不能把决策外包给"大神推荐",因为别人的场景和你不一样。

但这也很爽。

因为当别人还在争论"哪个 AI 更强"时,你已经在用三个 AI 协同工作了。

明天你可以这样做

如果你想复制 Chris 的思路,这里有三个具体行动:

1. 停止单一工具信仰,开始建工具矩阵

列出你现在用的所有 AI 工具,标注每个擅长什么。然后下次工作时,根据任务性质选工具——写文案用 Claude,深度研究用 ChatGPT,快速原型用 v0。




2. 打开 Plan Mode,给 AI 思考时间

不管用 Cursor 还是其他工具,遇到复杂任务时,先让 AI 做计划,你审核后再执行。这 30 秒的等待,能帮你避免 30 分钟的返工。




3. 研究你工具的隐藏功能

花一小时看文档、逛社区、问 AI:"这个工具有哪些高级用法?"你会发现很多惊喜——就像 Chris 发现 ultrathink 一样。

最后,给你一个决策框架:

低风险、高频率的任务(如 UI 调整、文案修改)→ 快速迭代,用最顺手的工具

高风险、低频率的任务(如架构设计、安全配置)→ 深度规划,用最强的模型

卡住的任务(如奇怪的 bug)→ 换个工具,换个角度
