今天 AI 圈有个新玩具 PixVerse 刷屏了。

是一个很神奇的东西：

AI 开始实时生成一个世界。

简而言之，就是可以通过 prompt 实时改变视频内容




一、以前的 ai 视频

过去我们玩视频生成，大概是这个流程：

写 prompt → 点生成 → 去倒杯水 → 回来看片

PixVerse 不一样。
它更像是：

你说一句 → 世界马上回应一句
你改一句 → 世界当场变样

就是一边听你说，一边搭建世界

然后我就贼好奇一个问题，

如果不让人来写 prompt，会发生什么？



二、于是我干了一件有点奇怪的事

我没有自己写 prompt。
我让 Claude Code 来写。

直接上图

流程非常简单，

CC 负责生成 prompt

也控制浏览器，实时把 提示词输入给 PixVerse

我只负责看结果，啥都不纠正

换句话说，这是一次对 AI 潜意识的旁观。

当 AI 没有讨好人类需求时，它会生成什么样的世界？

# PixVerse R1 实时世界生成
通过 agent-browser 控制 PixVerse R1，AI 生成世界观并实时演绎故事。
## 依赖
```bash
npm install -g agent-browser
agent-browser install
```
## 工作流程
### 1. 生成世界观
当用户请求时，生成一个富有想象力的世界观描述（50-100字），例如：
> 一座被遗忘在沙漠中的废弃游乐园，夜空繁星闪烁，生锈的摩天轮缓缓转动，霓虹灯若隐若现地闪烁。
### 2. 等待人类操作
告知用户执行以下步骤：
1. 打开 https://realtime.pixverse.ai/discover/
2. 点击 Create World
3. 手动输入世界观
4. 点击 Start Experience
5. 说"开始了"
### 3. 接管实时输入
当用户说"开始了"，执行：
```bash
# 获取元素
agent-browser snapshot -i
# 预期：输入框 [ref=e5]、Send 按钮 [ref=e7]
# 输入场景并发送
agent-browser type @e5 "场景描述（中文）"
agent-browser click @e7
# 等待 10 秒后继续
agent-browser wait 10000
```
循环输入场景，持续约 5 分钟。
## 场景创作原则
- 使用中文描述
- 围绕世界观展开
- 包含视觉元素：光影、天气、动作、氛围
- 每个场景 15-30 字
- 场景间有连贯性
## 示例场景序列
世界观：废弃游乐园
1. 月光洒在生锈的摩天轮上，轮子开始缓缓转动

3. 旋转木马的彩绘马匹眼睛闪烁，仿佛活了过来
4. 远处传来悠扬的音乐盒旋律
5. 一颗流星划过夜空，照亮废弃的过山车轨道
