# 别羡慕 Claude 了！Codex 史诗级更新，Skills 3 步给 AI 装上“外挂大脑”！

> 作者: 灵析探猿Everyday | 发布日期: 2025年12月18日 14:33
> 原文链接: [点击查看](https://mp.weixin.qq.com/s/1dky1jNx9dHHLfni5iHAjQ)

---

一直以来，Claude Code 的 "Skills" 功能都是其杀手锏之一，让 AI 能够“记住”项目特定的知识和流程，这让不少 Codex CLI 的用户羡慕不已。

不过，这种“羡慕”可以停止了。

最近，OpenAI 的 Codex CLI 终于悄悄跟进，推出了完全对标 Claude Code 的 Skills（技能系统）。这意味着 Codex 用户现在也能拥有一个懂项目、懂规范、懂流程的专属 AI 助手了。

今天我们就来深度解析这个“迟到但真香”的功能，看看它如何让你的 Codex 进化成完全体。

为什么说它是“外挂大脑”？



它就像是给 AI 助手安装了一个可插拔的**“外挂大脑”**。你可以把项目规范、常用流程、排错指南打包成一个个“技能”，AI 就能自动读取并记在心里，随时待命。

它的聪明之处：像查图书馆索引一样工作



Skills 的工作原理非常精妙，它完美平衡了“知识量”和“记忆负担”。

技能仓库：你把所有技能文件（SKILL.md）放在 ~/.codex/skills/ 目录下。
启动扫描：Codex 启动时，会快速扫描这个目录。
建立索引（关键点！）：它不会把所有死记硬背到脑子里（那样会瞬间撑爆宝贵的 Context 上下文窗口）。相反，它只记住了技能的名字和简介。

这就好比通过图书馆的索引找书： AI 脑子里只存了一份“书目清单”。当你问“怎么提交代码？”时，它会根据清单检索，发现有一个叫“Git 提交规范”的技能似乎对口，这时它才会真正把那本书（技能详情）从书架上取下来细看。

这种“懒加载”机制意味着，你可以给它装成千上万个技能，而不会拖慢它的反应速度。

如何开启这个“外挂”？



方法一：检查当前状态

codex features list
# 看看输出里 skills 是不是 true


方法二：想尝尝鲜（单次启用）

codex --enable skills


方法三：永久开启（强烈推荐）

在 ~/.codex/config.toml 配置文件中添加一行：

[features]
skills = true


💡 关键提示：description（描述）字段是重中之重！还记得上面的“图书馆索引”比喻吗？这个描述就是索引卡片上的内容。AI 仅凭这个描述来判断是否需要调用该技能。如果描述写得含糊不清（比如只写个“规矩”），AI 可能永远都不知道什么时候该用它。

3. 重启 Codex

现在，下次你让它“帮我提交代码”时，它会自动遵循这些规则，仿佛它一直是你们团队的老员工。

为什么两大巨头都在做这个？



Claude Code 做，Codex CLI 也跟进，这不是巧合。背后反映了 AI 编程工具的一个关键技术趋势——从单纯的“大模型”转向“上下文经济学”。

突破 Token 瓶颈的“懒加载”哲学随着项目越来越大，你不可能把所有的 API 手册、过往 Bug 记录、架构文档都塞进每一次对话里（既贵又慢，还容易超出 Context Window 上限）。Skills 本质上是一种 RAG（检索增强生成）的微型化实践。它让 AI 能够“携带”无限的知识库，但只在真正需要时消耗有限的算力。

从“通用工具”进化为“团队成员”以前的 AI 只是一个“聪明的路人”，懂 Python 但不懂“你们公司的 Python 规范”。有了 Skills，新入职的员工（或者新启动的 AI）只要同步了团队的 Skills 仓库，就能立刻掌握这几年积累下来的所有“潜规则”。

这意味着，未来的竞争不再是谁的模型参数更大，而是谁能更好地管理和调用这些私有知识。

总结



Codex CLI 的 Skills 功能不仅仅是一个简单的文档库，它本质上是一个低成本、高效率的知识注入机制。

虽然是“借鉴”了 Claude Code 的设计，但对 Codex 用户来说，这绝对是一个迟来的好消息。它填补了 Codex 在“个性化记忆”上的短板，让通用的 AI 变成了懂你、懂团队、懂项目的专属伙伴。

别犹豫了，花 10 分钟整理几个常用技能，你会发现你的 AI 助手突然“开窍”了。
