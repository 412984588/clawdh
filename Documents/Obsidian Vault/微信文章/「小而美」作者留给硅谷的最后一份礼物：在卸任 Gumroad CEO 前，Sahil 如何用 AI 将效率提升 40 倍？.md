# 「小而美」作者留给硅谷的最后一份礼物：在卸任 Gumroad CEO 前，Sahil 如何用 AI 将效率提升 40 倍？

今天我们来聊聊：

《小而美》作者在 Gumroad 内部「强推」的 40 倍效率提升「 AI 工作流」。




上周，我看到一条新闻，不大不小，但很值得玩味：

Sahil Lavingia 正式卸任 Gumroad CEO 一职。




或许你对这个名字还有点陌生，但在欧美创作者圈子里，Gumroad 是一个现象级的存在。




如果你想卖点什么，不管是你写的一本电子书、做的一套表情包、录的一门课，甚至是按月订阅的 Newsletter，Gumroad 几乎是你的首选。




Gumroad 售卖商品涵盖非常广。




它主打「你知道什么，就卖什么」，上传、定价、分享链接，钱就进账了。




截至目前，它已经帮创作者们卖出了超过 10 亿美元的商品。




但 Sahil 这位创始人留给行业的遗产，远不止 Gumroad 这个平台本身。尤其是在他卸任前推行的「40 倍效率提升 AI 工作流」改革，几乎可以看作是他在公司留下的最后一次、也最深刻的实验。




今天，我将通过具体的实测内容，带大家深入了解 Gumroad 的 AI 工作流和思路。

极简主义能有多极简？

下面，我想先讲讲 Sahil 这个人，真的还挺传奇的。




Sahil Lavingia 并不是一位传统的硅谷 CEO。




2011 年，19 岁的 Sahil 创立 Gumroad，那时候他像所有硅谷创业者一样，拿融资、招人、扩张，团队一度达到了 40 多人。




但后来，增长遇到了瓶颈。为了让公司活下去，Sahil 做了一个极其反直觉的决定：他没有以此为借口去融更多的钱，而是直接裁掉了 75% 的员工。




在公司年收入达到 1100 万美元的高光时刻，整个团队只有 25 人，而且几乎全是远程兼职。在很长一段时间里，Gumroad 甚至没有传统意义上的「全职员工」，全靠合同工和外包。




Sahil 甚至为此写了一本「圣经」：《The Minimalist Entrepreneur》（小而美：持续盈利的经营法则）。他的核心理念就是：剔除冗余，只保留核心价值。

但这还不够。




而当时间进入 AI 时代，Sahil 发现，实现「极简」的工具升级了。







前不久，Sahil 在做客 Lenny's Newsletter 时，分享了 Gumroad 在 AI 应用上的真实数据。这些数据没有水分，非常直观地展示了技术带来的变化：




（1）目前在 Gumroad 及其新项目 Flexile 中，41% 的代码合并请求（PRs）是由 Devin 编写的。Sahil 预计，到明年年底，这个比例将达到 80%。




（2）过去开发一个组件，人类工程师可能需要 2 周时间来处理各种边缘情况。现在通过 AI 工作流，从设计到代码落地，仅需 2 小时。这就是 40 倍的效率提升。




全面拆解：Gumroad 的「设计驱动开发」工作流

Sahil 将这套流程形容为一种「飞行员模式」：人类负责起飞（定义需求）、降落（验收质量）以及决定航向，而中间繁琐的飞行过程（编码与测试），大部分交给 AI。




这套工作流的核心工具栈非常精简：V0 + Devin + Cursor。




让我们深入这套工作流的三个核心环节，看看他是如何把 V0（设计）、Devin（开发）和 Cursor（微调） 串联起来的。




V0 (原型设计)： 首选的原型制作工具 。

Devin (AI 软件工程师)： 用于编写代码、测试和提交 PR 。

Cursor (IDE)： 用于手动微调或在 Devin 未能完成时进行补充 。

具体操作步骤：
高保真原型 (V0)：

📌

v0 是 Vercel 推出的一款生成式开发工具，在海外非常的火。




目的就是把「从零到上线」的开发流程简化到极致，直接通过自然语言生成可部署的应用。




简而言之，你只需要描述一下你想做什么（比如「一个有用户注册、仪表盘和图表分析功能的 web 应用」），v0 就会帮你自动生成前端代码（比如用 React 或 Next.js），并且自动生成样式（比如 Tailwind CSS），甚至还能帮你搭建一部分后端逻辑。




完成后，你只需一键通过 Vercel 部署，就能直接上线运行。




Link：https://v0.app




通过 Vercel V0:

以前： 产品经理写文档 -> 设计师画图 -> 反复沟通。现在： Sahil 直接使用 Vercel 推出的生成式 UI 工具 V0。




他不会只丢一句简单的指令，而是会花费 20-40 分钟 与 V0 进行多轮对话。




他的目标不是生成一个草图，而是打磨出一个包含完整 UI 细节和交互逻辑的高保真原型。

生成代码 (Devin)：

📌

Devin，你可以把它看成一个「AI 软件工程师」，由 Cognition Labs 开发的自主  AI Agent。




不仅仅是帮你写几行代码，而是能参与一个工程项目、规划、编码、测试、修复，再交付一套可运行的代码。




Link：https://devin.ai




通过 Devin：

以前： 工程师看着设计图，一行行敲代码。


现在： 将 V0 生成的设计链接或 Prompt 直接发给 Devin。




使用 Devin 后，开发效率会大幅提升，特别是当你面对那些「重复」或者「结构化」的任务（比如迁移、重构、修复 bug）时。




整个项目从「想法」到「代码＋测试＋部署」的过程变得更自动化，减少了很多中间的人力环节。




Devin 允许选择一个 Git 提供商，从 Git 仓库导入现有项目。







同样地，Devin 的交互方式也是「利用自然语言」对话，构建任务项目：







之后，Devin 会开启长长的自主任务工作流：




人工审查与合并 (Cursor)：

📌

Cursor 就不用多说了，它是目前最火的 AI IDE。




Cursor 是由 Anysphere 开发的一个 AI 驱动的代码编辑器，也可以算作是集成开发环境（IDE）工具。




它是基于 Visual Studio Code 的一个「分支」版本，整合了 AI，帮助开发者更高效地编程。




支持 Windows、macOS 和 Linux 三大平台，也基本覆盖了主流操作系统。




它的关键功能包括：通过自然语言生成代码、智能代码补全、支持多文件/项目的自动化重构，还能理解和查询整个代码库，能较大地提升开发效率。




Link：https://cursor.com




通过 Cursor：

以前：资深工程师需要花大量时间做 Code Review，还要帮新人改低级错误。

现在：人类回归到了QA（质量保证）和架构师的角色。




当 Devin 提交代码后，Sahil 会进行最后的把关。如果有一些极其复杂的逻辑 Devin 没处理好，他会使用 Cursor（AI 编辑器）进行手动微调。




这个环节的核心是「确认安全」。




就像机长在降落前检查仪表盘一样，人类确认架构合理、逻辑无误后，按下「合并代码」的按钮。




「杉」的观察：

在实战中，Sahil 甚至摸索出了一些看似老派却极其实用的小技巧：在 Prompt 中使用大写字母来强调重要指令（虽然有点老派，但很有效） 。




在列举例子时，只给出一两个例子然后加上 「etc」，可以让 AI 发挥创造力，进行即兴创作 。




当然，Sahil 留下的不只是这些「实操技巧」，更有一套完整的「AI 未来观」。




他对 AI 将如何重塑软件工程、组织结构以及未来的交付标准，有着很多有穿透力的洞察。




我总结了 4 点：

A. 工程角色的地壳运动 (Shift in Engineering Roles)

很多人担心「工程师会消失」，但 Sahil 的实践告诉我们，工程师不会消失，而是在「移位」。




人类工程师的重心，将从「写业务代码」转移到「修路」。他们负责搭建基础设施、消除技术债务，确保路面平整，以便 AI 能顺畅地跑起来。




他们可以直接通过 V0 等工具发布功能，不再需要苦苦等待后端工程师排期。




B. 迭代成本与「MVP 之死」 (MVP is Dead)

硅谷流行了十年的 MVP（最小可行性产品）概念，可能要过时了。




以前，工程师花两周写出的代码，如果被推翻，成本极高，大家会有心理抵触。




现在，AI 几分钟就能生成代码，推翻重来的成本趋近于零。这种「廉价的试错」鼓励了更大胆的尝试，从而带来更极致的用户体验。




正因为 AI 能快速产出高质量代码，粗糙的 MVP 已经不够看了。




Sahil 认为，开发者应该在 V0 阶段花费大量时间去打磨细节。因为 AI 拥有一种人类难以企及的特质：极高的责任心，它会不厌其烦地实现每一个你要求的微小细节。




C. 组织文化：用「真金白银」对抗恐惧

推行 AI 最大的阻力，往往不是技术，而是人性的恐惧。员工会本能地担心：「教会了 AI，是不是就要开除我？」




Sahil 解决这个问题的方法非常直接且粗暴：CEO 自己录视频演示怎么用 AI，自己提交代码（拿了第 4 名），证明这是赋能而非替代。




他设立了一个 3.3 万美元 的奖金池，专门奖励那些使用 Devin 最多的员工。




这种机制传递了一个明确信号：公司奖励「使用 AI 的人」，而不是「拒绝 AI 的人」。




D. 未来的 AI 形态：从被动到主动

目前的 AI 大多是「反应式」的（你问，它答）。但 Sahil 预判，未来的 AI 将变得极度「主动」。




想象一下，当你浏览网页时，AI 根据你的地理位置主动发起销售对话；或者当 GitHub 有了更新，AI 主动建议你发一条相关的营销推文。




在这种未来里，即便 AI 能做所有事，「决定做什么」（战略优先级排序）依然是人类不可被剥夺的核心价值。




Sahil 设想，未来 AI 甚至能根据销售数据，自动对功能开发列表进行排序，但最终拍板的，依然是人。







Sahil Lavingia 证明了一件事：在现代软件开发中，一个人的产出边界可以被极大地拓宽。




以前，我们需要一支庞大的团队来维持业务，因为沟通成本高、执行效率低。现在，通过合理的工具组合与组织变革，小团队甚至个人，也能维持令大厂汗颜的开发效率。




当我们不再被琐碎的代码细节捆绑，我们才能真正去思考那些更宏大的问题：

我们要创造什么？以及，为什么要创造它？
