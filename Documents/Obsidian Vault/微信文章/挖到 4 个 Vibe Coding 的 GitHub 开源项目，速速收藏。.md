# 挖到 4 个 Vibe Coding 的 GitHub 开源项目，速速收藏。

> 作者: 逛逛GitHub | 发布日期: 2025年12月12日 02:04
> 原文链接: [点击查看](https://mp.weixin.qq.com/s/Uj73FZc8yYusJAMy2aBY-w)

---

Andrej Karpathy 是 OpenAI 创始成员、前特斯拉 AI 总监。

他年初在 X 上发布了一条推文，分享了自己使用 Cursor + Claude 进行编程的体验，并首次使用了 Vibe Coding 这个词。

Vibe Coding 是一种全新的、基于 AI 的编程方式。你通过对话的方式和 AI 协作，开发出一个应用或者网站。这个过程甚至忘记代码的存在。

你只负责看效果，如果不对就继续对 AI 说，直到感觉对味了为止。


所以你可以完全专注于功能实现，而忽略底层代码细节。相当沉浸、相当 Vibe。

今天推荐几个 GitHub 上 Vibe Coding 相关的开源项目，感兴趣的速速收藏。

01

AI 编程指南

如果你没有任何基础，也想试一下 Vibe Coding 但是不知道怎么搞。

建议先把这个叫 aicodeguide 的项目内容搂一遍。

这是一份 AI 编程路线图，帮助零基础小白适应 AI 时代的编程新范式。

写的很详细，系统整理了 Cursor、Bolt.new 等前沿 AI 工具栈，也详细拆解了如何通过 Prompt 高效控制 AI 完成开发的具体工作流与最佳实践。

同时还有各种资源、教程和博客，如果你从来没搞过 Vibe Coding ，从这个开源项目开始吧。

开源地址：https://github.com/automata/aicodeguide

02

提示词模板

这个开源项目帮你梳理了一套工作流程和提示词模板，能让你在数小时构建一个产品 MVP 版本。

这个项目将 AI 开发软件的过程拆解为 5 个标准步骤：Research（研究）、Define（定义）、Design（设计）、Generate Agent Instructions（生成指令）、Build（构建）。


并为每一步提供了专门的提示词模版，以确保 AI 输出的质量和一致性，大大提高了复杂项目的成功率。

这个开源项目会强制你在写代码前先生成 PRD 和技术文档，这虽然是传统工程实践，但在 AI 时代通过 Prompt 自动化后变得非常高效。

如果你是产品经理或创业者，想要快速验证想法、构建 MVP，就来试试这个开源项目。

开源地址：https://github.com/KhazP/vibe-coding-prompt-template

另外如果你使用 Claude Code 作为主力工具，可以看看下面这个项目。


这是一套配置和指令的集合，可以把里面的配置逻辑复制到你的全局配置或项目根目录的配置中。

这样你就能把 Research -> Plan -> Code 的理念固化到了工具的默认设置里了。

开源地址：https://github.com/feiskyer/claude-code-settings

03

Vibe Coding 终极指南

如果你想用 Claude Code 或 Cursor 来 Vibe Coding，这个指南有两个版本，可以分别看 V1.2 和 V1.1。

这个作者的理念是：规划就是一切。 不要让 AI 自主规划，否则你的代码库将变得一团糟。

这个开源项目侧重于教育和流程引导。

作者不仅提供了 Prompt，还通过详细的文档手把手教你如何配置环境、选择工具、以及如何用 Prompt 把控开发节奏。

开源地址：https://github.com/EnzeD/vibe-coding

04

Awesome Vibe Coding

汇总全球范围内各类 AI 辅助编程工具。 

它不只收集提示词，更多的是收集软件工具。如果你想知道除了 Cursor 和 Windsurf 之外，还有哪些冷门但好用的 AI 编程工具，看这里就对了。

该项目主要包含以下几大板块：IDE 与插件、命令行工具、网页版工具、相关教程资源。

看了一下，网页版工具还有挺多有意思的开源项目的，逛逛后面单独发一篇文章介绍一下。

开源地址：https://github.com/filipecalegario/awesome-vibe-coding
