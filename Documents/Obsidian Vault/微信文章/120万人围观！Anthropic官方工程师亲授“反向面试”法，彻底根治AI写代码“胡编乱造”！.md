他揭示了一个让我们这些“非程序员”听了想拍大腿的真理：“想让 AI 写出完美的代码，第一步不是给它下命令，而是让它‘闭嘴’，先来面试你。”
看到这个方法，我瞬间悟了。


这不就是我们土木工程里的“先出图纸，再施工”吗？


99%的人用AI编程，都是上来就喊：“给我盖个房子（写个网站）！


”结果呢？AI 既没问你要多大面积，也没问你地基情况，稀里糊涂盖了一栋歪楼。最后你还得花3倍的时间去修补、去加固。


今天，我就带大家复刻这个神技。不懂代码没关系，只要你会回答问题，你就能做出专业级的产品。


我也将这一条加入到我的AI编程心法中，这个真是一个好东西。


而这个技巧不止在AI编程，其实每个AI任务都需要先用到这个东西。
核心观点：AI编程如同土木工程，先设计图纸（Spec），再动工（Code），AI应先“面试”用户挖掘需求。
一
核心心法：慢即是快
❌ 错误的姿势（大多数人）：
1.写个简单的 Prompt：“帮我做一个自动填表的网页。”
2.AI 立刻开干，狂写代码。
3.结局： 数据库选错了、界面丑得要死、流程逻辑全是漏洞。你不得不又要改代码，又要修Bug，最后烂尾。（土木话翻译：不仅没图纸，连底勘都没做就敢打桩，这楼必塌。）
✅ 正确的姿势（官方推荐）：
口诀：Interview first. Spec second. Code last.（先面试 → 再出方案书 → 最后敲代码）
为什么这一招神？
Thariq 利用了 Claude Code 的一个隐藏神技：AskUserQuestionTool。在这个模式下，AI 不再是一个只会搬砖的“实习生”，它摇身一变成了一位经验丰富的“产品经理”。它不会瞎猜你的心思，而是会通过 40+ 个深度问题 来“榨干”你脑子里的想法。
“你是给谁用的？企业还是个人？”
“数据要保密吗？存本地还是云端？”
“遇到断网情况，需要离线缓存吗？”
这就是“发现（Discovery）”。它帮你把那些你自己都没想清楚的隐性需求，全部挖出来，变成白纸黑字的“施工图纸”。
二
实操指南：四步复刻“反向面试”流
想体验这种“被AI大神带飞”的感觉吗？跟着我做，只需四步：
第一步：准备一块“空地”
在你的项目文件夹里，新建一个空的 Markdown 文件，命名为 SPEC.md（这就相当于我们要画的图纸）。哪怕里面只写一句话：“我要做一个面向土木人的施工日志生成器”。
第二步：下达“面试”指令 (The Magic Prompt)
在 Claude Code 的终端中，复制粘贴下面这段神级 Prompt。这段话就像一道符咒，强制 AI 停止蛮干，开始思考：
read this @SPEC.md and interview me in detail using the AskUserQuestionTool about literally anything: technical implementation, UI & UX, concerns, tradeoffs, etc. but make sure the questions are not obvious.  be very in-depth and continue interviewing me continually until it's complete, then write the spec to the file.
(大古注：这段话的意思是——“读一下这个需求文件，然后用提问工具疯狂面试我！技术、界面、难点啥都问，别问废话，问深一点！直到你觉得完全搞懂了，再把方案写进文件里。”)
第三步：接受 AI 的“拷问”
按下回车，神奇的事情发生了。Claude 进入了 Interview Mode（面试模式）。
它会开始向你抛出一系列选择题或问答题：
“数据库选 SQLite 还是 Postgres？（不懂？它会告诉你适合轻量级还是企业级）”
“界面风格要 Ant Design 还是 Tailwind？”
“鉴权方式是用微信扫码还是账号密码？”
你只需要在终端里做选择题，或者用大白话回答。
这个过程可能会持续十几分钟，直到 AI 彻底理解了你的意图。
第四步：生成“图纸”并施工
面试结束后，Claude 会基于你的回答，将那个原本空荡荡的 SPEC.md 改写成一份详尽的技术规格说明书。
关键的最后一步：Thariq 建议此时 结束当前会话（输入 /clear 或重启），然后开启一个新的会话，输入：
impl @SPEC.md
(意思是：按照这张图纸，开始施工！)
因为现在的上下文里包含了一份完美定义的文档，AI 写代码时将不再有幻觉，不再有错误的假设，一气呵成！
💡 大古的总结：这是非程序员的福音
为什么我强烈推荐土木人通过这种方式学编程？
1.决策前置 ：改文档里的文字是免费的，但改写好的代码是昂贵的。就像在纸上改图纸容易，等楼盖起来了再拆墙就难了。
2.收敛解空间 ：通过面试，你排除了无数种“AI 以为你想要但其实你不需要”的错误路径。
3.挖掘隐性需求：作为非专业人士，我们往往考虑不周全（比如并发问题、报错处理）。AI 会通过提问，帮你把这些坑提前填上。
在这个时代，写代码的能力越来越廉价，但“定义问题”和“设计方案”的能力越来越值钱。下次想做个工具时，别急着让AI写代码。先让它闭嘴，坐下来，好好面试你一番。
