# 我刚发现一个神仙项目，Claude Code 用户必看！

> 作者: 窦窦的干货库 | 发布日期: 2026年1月12日 09:28
> 原文链接: [点击查看](https://mp.weixin.qq.com/s/Oo3RqdZNEz_AL7RIiuV93w)

---

兄弟们，我今天刷 GitHub 刷到一个东西，直接给我看傻了——有人把 GPT 5.2 塞进 Claude Code 当小弟用了！

你没听错。就是那个 claude-delegator 项目。

这玩意儿到底是啥？

简单说：Claude Code 现在可以一键召唤 GPT 5.2 当专家顾问团了。

以前你想在 Claude Code 里用 GPT？得自己开 Codex CLI，各种折腾，麻烦得要死。现在呢？人家直接做成了原生 MCP 插件，Claude 动动嘴皮子就能调度一整个 GPT 专家团队。

让我拍大腿的点来了

这不是单纯的「调用另一个模型」，而是——

每个 GPT 子代理都有专精方向：有的专门搞架构设计，有的专门做安全审计
能给建议，也能直接上手改代码：不是那种只会 BB 的顾问，是真能干活的
通过 MCP 原生集成：丝滑得像 Claude 自己的能力一样

你想想这个场景：你让 Claude 写代码，遇到安全问题，Claude 直接甩给 GPT 5.2 安全专家审一遍，审完直接改。这不就是 AI 版的「专家会诊」吗？

我为什么觉得这事儿很大

说实话，看到这个我第一反应是：AI 协作时代真的来了。

以前我们总纠结「用 Claude 还是用 GPT」，现在人家的思路是——全都要，而且让它们互相配合。Claude 当主脑调度，GPT 当专项顾问，各司其职。

这让我想起一句话：未来最强的不是单个 AI，而是会调度 AI 的 AI。

用 Claude Code 的朋友，这个真的建议去看看。 哪怕不用，了解一下这个思路也值了。

链接我放这儿了：https://github.com/jarrodwatts/claude-delegator
