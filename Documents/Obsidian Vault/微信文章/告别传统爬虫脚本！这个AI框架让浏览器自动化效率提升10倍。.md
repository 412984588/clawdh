# 告别传统爬虫脚本！这个AI框架让浏览器自动化效率提升10倍。

> 作者: 猫说AI | 发布日期: 2025年10月9日 20:03

项目速览

Stagehand 是一个革命性的 AI 浏览器自动化框架，它完美结合了 Playwright 的稳定性和 LLM 的智能，让你能用自然语言指挥浏览器进行复杂操作，彻底告别传统自动化脚本的编写困境。

对于我们这些写过 Selenium 或 Playwright 脚本的程序员来说，最痛苦的就是元素定位和处理复杂交互。而 Stagehand 的目标很简单：让你选择什么时候写代码，什么时候动嘴（自然语言）。

这是提升效率的利器，确实值得一试。

快速上手：三步让你跑起来

Stagehand 的上手难度几乎为零，主要基于 Node.js/TypeScript 环境。它甚至提供了一个一键创建项目的 CLI 工具。

第一步：初始化 Stagehand 项目

打开你的终端，使用 npx 快速启动一个 Stagehand 模板。

npx create-browser-app

这个命令会帮你设置好所有的依赖和配置文件，包括 Playwright。

第二步：配置 LLM 和浏览器服务

Stagehand 依赖大型语言模型（LLM）来理解你的自然语言指令，并且推荐使用 Browserbase 来处理无头浏览器环境。

你只需要复制 .env.example，然后填入你的 LLM API Key 和 Browserbase 凭证即可。

cp .env.example .env
# 编辑 .env 文件，添加你的 API Key
第三步：开始混合编码

现在，你可以在项目中开始编写第一个混合脚本了。Stagehand 提供了两种编写风格：纯代码（Playwright）和 AI 驱动（自然语言）。

功能演示：代码与指令的丝滑切换

Stagehand 最强大的地方在于它的混合编程模型。你不需要完全依赖 AI 那个“黑盒子”，可以在关键节点使用稳定精确的 Playwright API，在需要应对复杂或多变页面时，再把控制权交给 AI。

我们来看一个 GitHub 自动化演示，这个动图展示了 Stagehand 如何通过指令自动执行操作：

Stagehand Action Demo

核心能力主要体现在以下几个 API 上：

1. 自然语言操作（Act）

对于简单的点击、输入或跳转，不再需要查找复杂的 CSS 选择器。

// Stagehand 兼容 Playwright 的 Page 对象
const page = stagehand.page;
await page.goto("https://github.com/browserbase");

// 使用 act()，直接告诉 AI 要做什么
await page.act("click on the stagehand repo");
2. 高级 Agent 任务（Execute）

如果需要完成一个涉及多步骤的复杂任务，比如“找到最新的 Pull Request 并进入”，你可以调用高级 Agent 模型。

Stagehand 允许你直接集成 OpenAI 或 Anthropic 最新的“计算机使用模型”（Computer Use Agents），让浏览器真正具备“自主操作”的能力。

// 引入 Agent 模型，执行复杂流程
const agent = stagehand.agent({
    provider: "openai",
    model: "computer-use-preview",
});

// 一条指令，让 Agent 自主规划路径
await agent.execute("Get to the latest PR");
3. 结构化数据提取（Extract）

爬虫最核心的需求就是提取数据。Stagehand 结合了 Zod 这样的 Schema 库，确保 AI 提取的数据是精确且结构化的 JSON 格式。

// 使用 extract() 提取页面信息，并定义好数据结构
const { author, title } = await page.extract({
  instruction: "extract the author and title of the PR",
  schema: z.object({
    author: z.string().describe("The username of the PR author"),
    title: z.string().describe("The title of the PR"),
  }),
});
// 结果直接是 TypeScript 对象，类型安全！

这种开发体验，简直就是降维打击，尤其对于需要频繁维护自动化脚本的业务来说，效率提升不是一星半点。

项目技术栈

Stagehand 采用 TypeScript 编写，并以 Playwright 作为底层骨干。这是它能保证高可靠性的关键。

Playwright 提供了坚实、稳定的浏览器控制能力，它负责执行最终的点击和输入操作。

LLM/AI Agent 扮演的是“智能指挥官”的角色。当开发者输入自然语言指令后，Stagehand 会通过 LLM（如 OpenAI 的视觉模型）来理解页面状态，并决定下一步应该在 Playwright 中执行什么操作。

这种架构避免了纯 Agent 方案中常见的“盲目”和“不可预测”的问题。开发者可以自由切换，保证了自动化脚本既智能又稳定。

此外，虽然 Stagehand 主版本基于 TypeScript，但项目方也提供了独立的 Python 实现，方便 Python 生态的开发者使用。

Stagehand 这个框架，让我看到了浏览器自动化未来的趋势。开源，免费，且功能强大。如果你还在为重复的网页操作或复杂的爬虫定位犯愁，Stagehand 绝对值得你认真研究一下。
