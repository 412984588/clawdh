Skyvern：用AI解放你的浏览器自动化工作
前言

你是否曾经为了重复填写表单、批量下载发票、或是在多个网站上完成采购而感到痛苦？传统的浏览器自动化工具虽然能帮忙，但每当网站更新布局，脚本就会失效，需要重新维护。今天要介绍的 Skyvern，一个基于大语言模型（LLM）和计算机视觉的开源浏览器自动化工具，彻底改变了这个游戏规则。

一、解决了什么问题？
1. 传统自动化的痛点

传统的浏览器自动化方案（如 Selenium）依赖于 XPath 和 DOM 解析来定位网页元素。这种方式存在明显的缺陷：

• 脆弱性：网站布局一旦变化，自动化脚本立即失效
• 定制化：每个网站都需要编写专门的脚本
• 维护成本高：需要持续跟进网站更新并修改代码
• 扩展性差：难以将一个脚本应用到多个相似网站
2. Skyvern 的革命性突破

Skyvern 通过引入 AI 技术，实现了质的飞跃：

• 视觉理解：像人一样"看"网页，而不是依赖固定的选择器
• 自适应能力：首次访问的网站也能自动操作
• 抗变化能力：网站布局改变不影响自动化流程
• 通用性：一个工作流可以应用到多个网站
二、Skyvern 是什么？
核心技术

Skyvern 是一个开源的浏览器自动化平台，它结合了：

1. 大语言模型（LLM）：理解任务意图和上下文
2. 计算机视觉：识别页面可交互元素
3. 代理架构：采用任务驱动的自主代理设计
4. 浏览器控制：基于 Playwright 实现真实浏览器操作
工作原理

Skyvern 使用一群智能代理（swarm of agents）来完成任务：

1. 理解网站：通过视觉模型分析页面结构
2. 制定计划：根据任务目标规划操作步骤
3. 执行操作：像人一样与网页交互
4. 自我修正：遇到问题时调整策略

支持的 LLM 包括 GPT-4o、Claude 3.5 Sonnet、Claude 4 系列等多模态模型。




三、怎么用？
1. 快速开始

环境要求：

• Docker Desktop
• Python 环境
• UV 包管理器

本地部署：

# 创建虚拟环境
uv venv

# 启动服务
docker-compose up

# 访问 UI 界面
http://localhost:8080
2. 核心功能
（1）表单自动填写

通过 navigation_goal 参数传递信息，Skyvern 会自动理解并填写表单：

{
  "url": "https://example.com/form",
  "navigation_goal": "填写联系表单：姓名-张三，邮箱-zhangsan@example.com"
}
（2）数据提取

从网页中提取结构化数据，支持复杂布局：

{
  "url": "https://example.com/products",
  "data_extraction_goal": "提取所有产品的名称、价格和库存信息"
}
（3）工作流编排

创建多步骤工作流，例如电商购物：

1. 导航到产品页面
2. 添加商品到购物车
3. 验证购物车状态
4. 完成结账流程
（4）实时视频流

通过浏览器实时观看 Skyvern 的操作过程，便于调试和干预。
