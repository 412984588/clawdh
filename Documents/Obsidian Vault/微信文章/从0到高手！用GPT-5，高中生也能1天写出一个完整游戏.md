第一章：入门

主工具选择：要开始 Vibe Coding，只需要以下工具之一：

• Claude Sonnet 4.5 （Claude Code）
• gpt-5-codex (high) （Codex CLI）

工具版本支持：支持 CLI（终端版本）和 VSCode 扩展版本。

历史工具演变：从早期的  Grok 3  转变为  Gemini 2.5 Pro ，现推荐使用最新的  Claude 4.5  或  gpt-5-codex (high) 。

重要原则： 计划是关键。不要让 AI 自动规划，否则代码库将难以管理。

第二章：环境搭建
1. 游戏设计文档

用  GPT-5  或  Sonnet 4.5  创建游戏设计文档 game-design-document.md，确保文档简明扼要并与您的需求一致。

文档目的是为 AI 提供有关游戏结构与意图的清晰上下文。

2. 技术栈和规则文件（CLAUDE.md / Agents.md）

让  GPT-5  或  Sonnet 4.5  推荐最优游戏技术栈，并保存为 tech-stack.md。

在  Claude Code  或  Codex CLI  中运行 /init 命令，基于 game-design-document.md 和 tech-stack.md 生成规则。

重点检查生成的规则，确保它们突出模块化（避免写单一巨型文件）。

关键规则示例（标记为“Always”）：

# 重要：
# 在写任何代码前，请始终阅读 memory-bank/@architecture.md，包括整个数据库架构。
# 在写任何代码前，请始终阅读 memory-bank/@game-design-document.md。
# 在完成一个主要功能或实现里程碑后，更新 memory-bank/@architecture.md。
3. 实施计划 (implementation-plan.md)

向  GPT-5  或  Sonnet 4.5  提供游戏设计文档和技术栈文档，并生成详细的分步实施计划。

• 每一步都包括具体任务和测试来验证正确性。
• 专注于构建基础游戏，而不是添加所有功能。
4. 创建 Memory Bank

在项目文件夹中新建子文件夹 memory-bank，并添加以下文件：

• game-design-document.md
• tech-stack.md
• implementation-plan.md
• progress.md（空文件，用于跟踪已完成步骤）
• architecture.md（空文件，用于记录架构内容）
第三章：基础游戏的 Vibe Coding
准备工作

启动  Codex  或  Claude Code  的 VSCode 扩展或终端版本。

检查 implementation-plan.md 是否清晰，若存在问题，回答 AI 的问题并适时修改。

第一步实施

提示 AI：完成 implementation-plan.md 的第1步，运行测试，等待用户验证。

在用户验证后，更新 progress.md（记录完成情况）和 architecture.md（说明文件用途）。

流程循环

完成一个步骤后通过 Git 提交变更。

重启对话并推动 AI 进行下一个步骤，确保逐步推进实施计划。

沉浸式操作技巧

使用 Superwhisper 通过语音交流 AI，而无需键盘输入。

第四章：细节扩展

构建基础游戏后，可逐步添加功能（例如迷雾、后处理或特效）。

为每个重大功能创建一个新的 feature-implementation.md 文件，逐步测试实现。

第五章：问题解决与调试
提示失败或代码崩溃

使用 Claude Code 的 /rewind 命令回滚。

提供错误截图帮助 AI 改进。

如果项目卡住，可用 RepoPrompt 或 uithub 将代码整合并请求 AI 帮助。

最佳实践

避免信息过载，利用 /clear 或 /compact 清除上下文。

使用 claude --dangerously-skip-permissions 或 codex --yolo 跳过确认以加速开发。

第六章：工具与提示技巧

小型修改：  使用 GPT-5（medium）。

创意营销文案生成：  使用 Opus 4.1。

生成精美的 2D 图片：  使用 ChatGPT 和 Nano Banana。

音效生成：  使用 ElevenLabs。

视频生成：  使用 Sora 2。

更优化的提示：  明确指令并强调 AI 深度思考（如“保持精准并完美执行”）。

第七章：常见问题
游戏与应用开发流程差异

开发应用与游戏流程基本相似，使用 PRD（产品需求文档）替代 GDD（游戏设计文档）。

驱动高质量结果的具体提示示例

使用清晰简短的提示，例如“在机翼上开挖空间用于副翼”，避免模糊指令。

Claude Code 或 Codex CLI 的优势

VSCode 的终端版本支持广泛的开发工作流（如远程服务器 SSH），并通过定制命令和子代理提高开发效率。

多人游戏服务器设置

直接向 AI 询问具体服务器搭建问题。
