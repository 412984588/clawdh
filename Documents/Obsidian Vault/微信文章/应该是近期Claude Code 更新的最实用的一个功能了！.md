# 应该是近期Claude Code 更新的最实用的一个功能了！

> 作者: 字节笔记本 | 发布日期: 2025年12月16日 04:38
> 原文链接: [点击查看](https://mp.weixin.qq.com/s/ORvEgptgBT3kl1dCJ4su1w)

---

这应该是 Claude Code 近期最实用的一个更新，没有之一。

在这个功能发布之前，我们使用 AI 编程的方式更像是抽卡。

输入一个复杂的指令，祈祷 AI 能一次性理解你的意图，理清上下文，并且写出不破坏现有架构的代码。

对于现在的模型能力来说，目前还处于一个初始阶段，一个是模型的上下文有限90%的人都选错了！Claude Code Skills 到底选哪个当脚本？，二来是模型幻觉太多，200k Tokens 的上下文真的够用吗？

Claude Code  feature-dev的出现，标志着 Claude Code 从一个 编程助手进化为了一个工程团队。

它不再只是听指令写代码，它开始有了 SOP 标准作业程序，以前我们追求快，现在 feature-dev 告诉我们，慢就是快。

目前这个功能Claude Code默认不提供，需要我们安装官方市场。

/plugin marketplace add anthropics/claude-code

/plugin install feature-dev


输入 /feature-dev 启动任务后，Claude 不会急着写下第一行代码。

它会强制启动一个包含 7 个阶段的瀑布流开发模式。

阶段一：反向面试

这是最让我惊喜的环节。

过去是我们向 AI 提需求。在 feature-dev 的第一阶段，角色互换了。

它会像一个负责任的产品经理一样，反过来向你确认需求。

要缓存什么内容？
性能指标是多少？
鉴权逻辑怎么处理？


它通过提问，把模糊的自然语言，逼近为精确的技术规格。

仅仅这一步，就能消灭了50%的返工风险。

阶段二与三：谋定而后动

在开始写代码之前，它会启动 code-explorer 智能体。

它会先去翻阅你现在的代码库，看看认证系统是怎么写的，看看数据库连接池是怎么配的，看UI组件是如何构成的。

然后 code-architect 智能体接手，它不写代码，只写方案。

它会告诉你，我准备在 src/api 下新增一个中间件，复用现有的 Redis 连接。

这就是高级工程师和初级码农的区别，初级码农上来就写，高级工程师先谋划后动手。

阶段四至六：自我闭环

进入编码阶段后，feature-dev 并不是写完就跑。

它内置了测试验证和代码审查环节。

code-reviewer 智能体会在最后一道关卡，检查代码标准遵从性。

不是简单的语法检查，而是逻辑审查。

feature-dev 的本质，是把 Agentic Workflow 智能体工作流 也就是我们常说的 Agent 模式，真正落地到了日常开发中。

尤其对于那些涉及到多个文件修改、需要遵循严格架构规范的复杂功能，它是神器。

从今天开始，强迫自己使用 feature-dev 处理所有超过 50 行代码变更的任务。

它在前期多花的每一分钟思考时间，都能在后期为你节省十分钟的 Debug 时间。
