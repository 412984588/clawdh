如何用 swiftUI 开发安卓应用？

说实话，自从上个月 Swift SDK for Android 正式发布之后，跨平台开发这个话题就越来越热了。很多独立开发者都在问，能不能用一套 SwiftUI 代码同时跑 iOS 和 Android？现在 Skip Fuse 免费了，这个问题算是有了个不错的答案。

如果你也在考虑做跨平台开发，或者对 Swift 在 Android 上的应用感兴趣，今天这篇应该能给你一些参考。

Skip 的两种模式：Lite vs Fuse

Skip 其实有两种模式，之前独立开发者只能用 Lite，现在 Fuse 也免费了。

Lite 模式：代码转换

Lite 模式说白了就是把你的 Swift 代码转换成 Kotlin/Java。这种方式的好处是实现起来相对简单，但缺点也很明显：转换后的代码可能不够优化，性能上会有一些损失。

我之前试过 Lite 模式，写个小 demo 还行，但真要做一个完整的应用，还是会担心性能问题。毕竟代码经过转换之后，编译器能做的优化就有限了。

Fuse 模式：原生编译

Fuse 模式就不一样了，它不会转换你的代码，而是直接把 Swift 编译成 Android 的原生 ARM 代码。这种方式的好处是性能更好，因为你的 Swift 代码是直接编译成原生二进制的，没有中间层的开销。

Fuse 模式的核心优势：

真正的原生性能：你的 Swift 代码直接编译成 ARM 代码，不需要 JavaScript 桥接（像 React Native 那样），也不需要自定义渲染引擎（像 Flutter 那样）。代码运行起来就是原生 Android 二进制，只有跟 Android Kotlin/Java 库交互的时候才需要桥接。

使用官方 UI 框架：这是最吸引我的一点。你可以用 SwiftUI 写 iOS 端的 UI，Skip 会自动生成对应的 Jetpack Compose UI 给 Android 端。这意味着用户看到的是平台原生的 UI，不是模仿出来的。而且你用的是苹果和谷歌官方推荐的现代 UI 框架，能享受到未来的平台更新和特性。

而且最重要的是，在 iOS 端，Skip 什么都不做。你直接用的就是苹果的框架，比如 SwiftUI，没有任何额外开销。这意味着用 Skip 做 iOS 应用，性能上完全不受影响。

如果你符合这些条件，现在就可以免费使用 Skip Fuse 了。之前只能用 Lite 模式，现在 Fuse 模式也免费了，这对独立开发者来说确实是个好消息。

说实话，3 万美元的年收入门槛，对很多刚开始做独立开发的开发者来说，应该不算太高。而且如果你做的是开源项目，完全不受限制。

适合什么样的项目

适合的项目：

新项目，从零开始。如果你要做一个新项目，可以考虑用 Skip Fuse，从一开始就做跨平台。

UI 为主的应用。如果你的应用主要是 UI 展示，业务逻辑不太复杂，Skip Fuse 应该能很好地满足需求。

独立开发者。如果你是一个人或者小团队，预算有限，Skip Fuse 免费了，可以试试。
