Claude前几天新出了个好东西，叫Skills，简单来说，Skills就像是给Claude加了一个插件，让Claude有了一些自定义的非常稳定可调用功能。

Skills




Skills是什么




它跟MCP不一样，MCP是调用了一个外部的接口去实现功能。




Skills是一个技能的文件，里面包含一些说明文件、脚本（代码）、所需资源。如果Claude识别到你当前的任务和某个技能相关，才会调用它，这样Claude就会更好的完成这项任务。




它让Claude不再是一个通用助手，而是可以加载不同的专业能力模块，还可以像调用MCP一样实现一些大模型本身无法实现的功能。




一个Skill就是一个能力包：




这个技能是什么

它能做什么（比如 讲解 API、调试错误、给出代码示例）

它该怎么做（工作流程）

它需要哪些资料（关联的文档、示例、模板）




Claude Skill = Claude的插件系统，结合工具调用 + 文档知识 + 结构化接口描述，让AI具备可组合的能力。




功能特点




可组合：多个技能可以一起用，Claude会自动选择需要的技能。

可复用：同一个技能可以在Claude App、Claude Code、API里通用。

轻量高效：只加载任务需要的部分，不会一次读完所有资料。

可执行：技能可以包含脚本或代码，Claude能直接执行任务。

可协作：技能支持版本管理和团队共享。

Skill Seekers







项目简介




Skill Seeker是一个能把任何文档网站自动转换为 Claude Skills的工具。它可自动抓取文档网站、按主题整理内容、借助AI模型提取关键概念，还能打包成可上传的 .zip 文件，支持PDF提取、本地增强等功能，原本需要几个小时的手动工作缩短至20 - 40分钟。




简单来说，实际上就是一个能力很强的爬虫，配合AI能力做成Skills，文档多的话自己手动复制粘贴都累死了，所以这个项目在效率上真的帮助很大。




而且Skills做的话也需要技巧，自己去研究不如直接用这个开源的工具更好，毕竟项目开发者都提前研究过了Skills怎么做效果更好。




功能原理




抓取（Scrape）：从文档中提取所有页面内容。


分类（Categorize）：将内容按主题组织，如 API、指南、教程等。


增强（Enhance）：AI 分析文档并自动生成完整的 SKILL.md，包含示例。


打包（Package）：把所有内容打包成可在 Claude 中直接使用的 .zip 文件。




功能特点




核心文档处理能力




多源内容支持：适配网页文档、PDF 文件，未来将支持 Word、Excel、Markdown。

智能抓取提取：自动抓取内容并按主题分类，支持代码块检测、表格提取、图像提取。

大型文档适配：处理10K-40K+页面文档，支持智能拆分、路由 / 枢纽技能功能。




AI增强功能




技能文档优化：将基础模板转化为500+行的全面指南，提取最佳示例并解释关键概念。

双增强模式：基于 Claude Code Max，免费无 API 成本，30-60秒完成。

API增强：使用Anthropic API，质量与本地版相当。

参考生成：自动提取常见代码模式，生成实用参考内容。




性能




并行处理：多线程抓取和PDF处理，减少66%总时间。

缓存机制：一次抓取可多次重建，重运行节省50% 时间。

断点续传：长时抓取任务支持断点恢复，避免进度丢失。




工具集成




MCP服务器：9个集成工具，支持通过Claude Code 自然语言操作。

CLI工具集：包含文档抓取、页面估算、技能打包等独立脚本。

预设配置：12 种验证配置，Godot、React、Vue等框架。




工作流




多模式操作：支持直接PDF模式、JSON模式。

批量处理：计划支持多站点 / 文件同时处理。

质量控制：内置质量评分（0-10分）、内容过滤和示例提取功能。
