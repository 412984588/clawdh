有了它：不写后端代码，也能拥有完整后端能力。

01

项目简介

AipexBase 是中国第一款开源 AI 原生后端即服务（Backend-as-a-Service）平台。 

它让你使用 AI 写应用的时候，无需关注后端接口开发。


数据存储、鉴权、三方接入、上下文管理等功能全部由平台自动封装，通过前端 SDK 或 MCP 即可一键调用后端能力。

无论你使用 Cursor、Claude Code 还是其他 AI 编程工具，都可以轻松接入 AipexBase 提供的 MCP，在 AI 辅助下直接完成从前端到后端的一体化开发。

02

如何使用

① 创建应用
部署好 aipexbase 会给你一个后台地址，可以进入后台管理页面创建一个应用，然后点击去创建一个 API key 复制保存好，后面配置 MCP 会用到。
② 配置 MCP
你可以在 Cursor 或者 MCP 客户端里面配置一下，需要替换一下 url 里面的域名和 token。
{
  "mcpServers": {
    "aipexbase-mcp-server": {
      "url": "http://你的域名或IP/mcp/sse?token=coding123"
    }
  }
}

如果刚刚安装的 MCP Server 的小绿灯亮了，就说明 MCP 配置成功喽。

然后在 Cursor 对话框内输入 / 即可唤醒并选择 aipexbase 内置软件研发提示词。
后面你就可以通过 AI 对话来开发一个包含前后端的应用了。后端交给 aipexbase ，快速实现用户登录、注册，数据量增删改成或调用 API 。
因为需要持久化保存数据，AI 会创建数据库。
这些新创建的数据库，会在你刚刚创建 token 的 aipexbase 应用中找到。

03

3 大亮点

AipexBase 这个开源项目有 3 大亮点。

① 真正的AI原生架构
AipexBase 从底层就是给 AI 应用设计的，原生兼容 MCP与 AI Coding 模式，这样模型和智能体就能直接调用后端能力了。
而且平台统一了上下文与数据层，让 AI 应用具备长期记忆与可追踪状态管理。这个特性对于需要持续学习和适应用户行为的 AI 应用来说，很重要。
② 国产化生态深度适配
看了下，与 Supabase 等国际产品不同，AipexBase 真正深度适配中国开发生态，名副其实的国产友好。
支持飞书、钉钉、微信这些等国内主流办公协同能力接入，同时兼容鸿蒙应用、小程序、WebView 容器。
这个还挺给劲儿的。
③ 开源开放
AipexBase 已经开源了，你可以根据需要自由定制和扩展平台能力。快去瞧瞧吧。
开源地址：https://gitee.com/kuafuai/aipexbase/tree/main  #GitHub开源链接在路上
