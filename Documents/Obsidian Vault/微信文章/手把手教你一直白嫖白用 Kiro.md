# 手把手教你一直白嫖白用 Kiro

一、Kiro 是什么？

Kiro 是一个在线 AI 工具平台，支持多种模型。
新用户注册后会送一部分试用额度，很适合先体验一段时间再考虑要不要长期使用。

本文主要教你：

用 Github 登录 Kiro
使用 临时邮箱 注册 Github
看清楚 试用额度 和 可用模型列表
文末附上 三个现成可用的测试账号
二、打开 Kiro 官网
浏览器输入网址：https://kiro.dev/
进入首页后，找到右上角的 「Login / 登录」 按钮并点击
三、选择 Github 登录方式

点击登录后，Kiro 会让你选择登录方式。
我们这里选择 Github 登录：

点击 「Github」（或类似字样）
浏览器会跳转到 Github 授权 / 注册页面


四、用临时邮箱注册 Github 账号

如果你不想用自己的常用邮箱，可以用 临时邮箱 注册一个新的 Github 账号。

大致步骤：

打开任意一个 临时邮箱网站
生成一个新的邮箱地址
回到 Github 注册页面，填写：
Email：临时邮箱地址
Username：随便取一个好记的用户名
Password：设置一个密码（注意记住）
打开刚才的临时邮箱，查收 Github 验证邮件
输入邮件中的验证码，即可注册成功


五、授权 Github 关联 Kiro 应用

Github 账号注册完成后，页面会出现一个授权界面，提示：

某某 Github 账号将授权 Kiro 访问基础信息

你只需要：

确认是你刚注册的那个账号
点击 「Authorize」（授权）即可




授权成功后，浏览器会自动跳转回 Kiro，进入你的个人中心。

六、查看试用额度：500 + 50，一共 550 USED

第一次注册并成功关联后，Kiro 会给一部分试用额度。
例如我的账号显示：

每月额度：50
新账号赠送：500
合计：550 USED

日常写代码什么的，这个额度可以用挺久的。




七、试用用完怎么办？

如果你把这 550 USED 用完了，有两种思路：

正经路线（推荐）：
觉得好用的话，按平台规则充值 / 订阅，支持一下产品。
继续体验路线（限度自控）：
再生成一个新的临时邮箱
再注册一个新的 Github 账号
重复上面的流程，再体验一轮



温馨提示：
具体操作请自行把握，尽量遵守 Kiro 官方的使用条款和相关规则。

八、可用模型列表预览

下载Kiro的IDE 进入 Kiro 后，可以查看当前的 可用模型列表，大致可以看到：

支持了哪些模型
是对话模型、工具类模型还是其他类型
方便你按需选择体验(因为可以一直换号，我都是直接用opus)

十、验证码怎么获取？

登录 / 注册过程中，如果遇到需要邮箱验证码的情况：

系统会提示往对应邮箱发送验证码
你只需要到那个邮箱里把验证码取出来
再输入回页面即可完成验证

这里的操作方式是：

想用自己注册：
去临时邮箱网站接收验证码邮件
想用我给的两个账号：
需要验证码时，关注我，
然后去临时邮箱自助接码即可（按我给的邮箱，自己去查对应邮箱里的验证码）
十一、整个流程总结

简单回顾一下：

打开 https://kiro.dev/，点击右上角登录
选择 Github 登录
用 临时邮箱 注册一个新的 Github 账号
在 Github 页面点击授权，完成 关联 Kiro 应用
回到 Kiro 个人中心，看到试用额度：50 + 500 = 550 USED
打开 模型列表，选择自己想体验的模型用起来
试用完可以：
正常付费继续用，或
换一个临时邮箱再注册一个新号（自行把握）
懒得注册的话，可以直接用文末给出的三个邮箱 + 密码登录 Kiro 体验
遇到验证码问题，就去对应邮箱里收验证码即可
