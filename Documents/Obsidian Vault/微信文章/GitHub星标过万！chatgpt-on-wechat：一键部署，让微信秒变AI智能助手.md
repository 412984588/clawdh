chatgpt-on-wechat：让微信秒变AI智能助手的开源神器

想让自己常用的微信、企业微信或者飞书接入强大的AI对话能力吗？chatgpt-on-wechat（简称CoW）项目让你能快速在熟悉的社交和工作平台上部署智能聊天机器人，无论是个人娱乐、学习辅助还是企业客服，它都能大显身手。

项目介绍

chatgpt-on-wechat 是一个基于 ChatGPT 的智能聊天机器人项目，它通过 OpenAI API 生成对话内容，并利用 itchat 等库实现微信等平台消息的接收与自动回复。该项目旨在将先进的AI对话能力无缝集成到日常使用的通讯工具中。

这个项目因其丰富的功能、灵活的配置以及活跃的开源社区，在GitHub上获得了广泛的关注和认可。它最初支持微信个人号和微信公众号部署，并逐步扩展至支持企业微信、飞书、钉钉等多种平台，真正实现了AI助手的多端覆盖。

核心功能

chatgpt-on-wechat 集成了多种强大且实用的AI功能，使其远超一个简单的自动回复机器人。

多端部署与基础对话
多平台支持：项目支持将机器人部署在个人微信、微信公众号、企业微信、飞书、钉钉等多个平台。这意味着你可以根据使用场景灵活选择部署方式。
智能回复：在私聊及群聊中实现消息的智能回复。通过简单的配置，可以设定触发机器人回复的前缀，例如在私聊中以 "bot" 开头，或在群聊中@机器人。
多模型与上下文记忆：支持 GPT-3.5, GPT-4, Claude, 文心一言, 讯飞星火 等多种大语言模型，并能进行多轮对话记忆，为每个好友或群组维护独立的会话上下文，让对话更连贯、更智能。
多模态交互能力
语音识别：可识别处理语音消息，并通过文字或语音进行回复。项目支持 Azure, Baidu, Google, OpenAI Whisper/TTS 等多种语音模型，灵活性强。
图像生成与识别：支持根据文字描述生成图片（文生图），以及图生图（如照片修复）。你可以选择使用 Dall-E-3, Stable Diffusion, Midjourney, Vision 等多种图像模型来满足不同的创作需求。
强大的扩展能力
丰富插件：项目支持个性化插件扩展，目前已实现多角色切换、文字冒险、敏感词过滤、聊天记录总结、文档总结与对话等多种插件，极大地丰富了机器人的可玩性和实用性。
Tool工具：机器人可以与操作系统和互联网交互，支持最新信息搜索、数学计算、天气和资讯查询、网页总结等功能。这部分功能基于 chatgpt-tool-hub 实现，让机器人不再局限于静态知识。
知识库：通过上传知识库文件（如PDF、Docx、TXT等），可以自定义专属机器人。这个功能非常适合打造个人数字分身、特定领域知识库或智能客服，让AI更"懂"你。
使用方法
安装与部署

chatgpt-on-wechat 提供了多种部署方式，这里介绍两种主流的方案。

传统方式部署

环境准备：

确保你的 Python 版本在 3.7.1 到 3.9.X 之间（Linux环境建议使用3.7.X）。
前往 OpenAI注册页面 创建账号并获取 API Key。

安装步骤：

克隆代码：

git clone https://github.com/zhayujie/chatgpt-on-wechat
cd chatgpt-on-wechat/


安装核心依赖：

pip3 install -r requirements.txt


配置项目： 将根目录下的 config-template.json 复制为 config.json，并填入你的配置。以下是一些关键配置项说明：

配置项
	
说明
	
示例

open_ai_api_key	
你的OpenAI API Key
	"sk-..."
single_chat_prefix	
私聊触发前缀
	["bot", "@bot"]
group_chat_prefix	
群聊触发前缀
	["@bot"]
group_name_white_list	
开启自动回复的群聊白名单
	["ChatGPT测试群"]
image_create_prefix	
触发图片生成的前缀
	["画", "看", "找"]
conversation_max_tokens	
上下文最大字符数
	1000
character_desc	
设定机器人的人格描述
	"你是..."

运行项目：

本地运行：直接在项目根目录执行 python3 app.py，扫码登录微信即可。
服务器部署：使用 nohup python3 app.py & tail -f nohup.out 命令在后台运行。
Docker方式部署

对于希望快速部署且环境隔离性要求高的用户，Docker方式是更好的选择。

下载配置文件：

wget https://open-1317903499.cos.ap-guangzhou.myqcloud.com/docker-compose.yml


修改配置：在 docker-compose.yml 文件中填入你的 OPEN_AI_API_KEY 和其他必要的配置项。

启动容器：

sudo docker compose up -d


查看日志并扫码登录：

sudo docker logs -f chatgpt-on-wechat


在日志中查看输出的二维码，使用微信扫码登录即可。

基本使用流程
部署成功后，你的账号就成为了一个AI机器人。
私聊：在微信中，对你部署为机器人的账号发送消息，当消息以配置的前缀（如 "bot"）开头时，机器人会进行回复。
群聊：在配置了白名单的群聊中，当有人@机器人或以配置的前缀（如 "@bot"）开头发送消息时，机器人会触发回复。
特殊功能：发送以图片生成前缀（如"画"）开头的消息，可以让AI为你创作图像。
技术架构

chatgpt-on-wechat 项目在技术实现上体现了模块化和可扩展的设计思想。

消息接收与分发：项目使用 itchat（针对微信个人号）或 wechatpy（针对企业微信）等库来接收和处理来自不同平台的消息。通过通道（Channel） 的概念，将不同平台的消息统一抽象和处理，使得支持新平台变得更容易。
核心处理引擎：接收到消息后，机器人会根据配置的规则判断是否触发回复。触发后，会将消息内容、上下文历史（如果配置了记忆）以及用户设定的人格描述一并发送给 OpenAI API（或其他支持的大模型API）。
插件与工具系统：项目支持插件化开发，允许开发者编写自定义插件来扩展机器人的功能。同时，通过集成 chatgpt-tool-hub，为机器人提供了访问外部工具和网络信息的能力，突破了纯文本对话的限制。

应用场景

chatgpt-on-wechat 的强大能力使其在多种场景下都能发挥作用。

个人娱乐与效率助手：作为个人微信好友，它可以陪你聊天解闷，回答各种问题，帮你总结文章、生成图片，甚至在你学习时充当辅导老师。
企业与社群管理：部署在企业微信或群聊中，它可以作为智能客服自动回答常见问题，提高服务效率；也可以在社群中活跃气氛，自动发布通知或总结聊天内容。

注意事项
账号风险：使用个人微信扫码登录存在一定的封号风险，需谨慎使用。建议使用小号进行测试，并关注项目社区的相关讨论。
API成本：使用OpenAI等API会产生费用，需要关注自己的API使用量和余额，合理配置对话字数限制等参数以控制成本。
网络问题：国内访问OpenAI API可能需要配置代理或使用反代地址。
