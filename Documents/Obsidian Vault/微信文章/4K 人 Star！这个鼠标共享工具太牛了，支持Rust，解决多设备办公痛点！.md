背景

想象一下，你的桌面现在有两台电脑，一台 Windows 主力机，一台 Mac 用于开发，甚至还有一台 Linux 服务器。你可能需要为每台设备都配备一套键盘鼠标，桌面变得无比凌乱。Lan-Mouse 就诞生于这样的痛点之中，旨在一劳永逸地解决多设备办公的键鼠共享问题。 它被誉为软件版的 KVM 切换器，让你的桌面瞬间清爽，效率飙升。

这个项目在 GitHub 上已累计 4K+ Star，社区反响热烈，被众多开发者誉为 “多设备生产力神器”。项目由个人开发者主导，采用 Rust 语言深度优化，确保了极致的性能和稳定性。其名称 “Lan-Mouse” 顾名思义，即基于局域网（LAN）的鼠标（Mouse）共享解决方案，简洁明了，直击核心。



01 项目简介

Lan-Mouse 是一个开源的跨平台键鼠共享软件，旨在让你只用一套键鼠，就能无缝控制局域网里的所有电脑。 其核心功能在于通过网络，将一套物理输入设备的信号，同步传输给多台不同的操作系统设备。无需代码，极低配置门槛，拖拽式的工作流搭建体验，让多设备协同前所未有的简单。

项目亮点：

在 AI 能力方面，Lan-Mouse 虽然不直接提供 AI 节点，但其无缝切换的体验，为多设备间的 AI 模型部署、数据处理工作流提供了坚实的基础。你可以轻松在一台设备启动 AI 计算任务，然后切换到另一台设备查看结果，多模态 Agent 的协作变得更加流畅。

关键特性：

支持 Windows, Linux, macOS 三大主流操作系统，尤其对 Linux 的 Wayland 环境进行了深度优化，解决了传统 X11 环境下的延迟问题。节点数量支持无限扩展，只要你的局域网带宽允许，即可连接任意多台设备。

其集成方案灵活，提供图形化用户界面（GUI）和命令行界面（CLI），满足不同用户群体的需求。

部署方式简便，支持跨平台安装包和源代码编译安装。

得益于 Rust 语言的高效实现，所有输入事件都通过 UDP 协议在局域网内传输，力求实现低延迟和快速响应。

采用 对等网络架构，增强了连接的稳定性和可靠性。



现代化、直观的用户界面，让配置和使用过程异常简单。



02 几个案例

1. 程序员的多屏幕开发环境


一个典型的开发场景，你可能有一个 Linux 主机作为开发服务器，一台 macOS 作为主工作站。

工作流步骤：



·在 Linux 服务器上运行 Lan-Mouse 服务端。

·在 macOS 工作站上运行 Lan-Mouse 客户端，并配置共享。

·通过简单的鼠标移动到屏幕边缘，即可将键鼠控制权无缝切换到 Linux 服务器，实现代码编写、命令执行。





效果： 极大地简化了多系统切换的繁琐，避免了反复插拔键鼠，提升了开发效率至少 30%。

2. 内容创作者的跨平台协作


一名内容创作者可能需要一台 Windows PC 进行视频剪辑和游戏，同时使用一台 Mac 进行平面设计。

工作流步骤：



 

·在 Windows 和 Mac 上都安装并运行 Lan-Mouse。

·配置主机和客户端。

·当在 Windows 上完成视频渲染后，直接将鼠标移动到 Mac 屏幕，继续进行图片素材的编辑。





效果： 实现了创作工具的无缝切换，减少了硬件投入，让创作者更专注于内容本身，而不是设备切换。

3. 家庭影音娱乐中心


家中有 HTPC (家庭影院电脑) 连接电视，还有一台日常使用的笔记本电脑。

工作流步骤：



·在笔记本上作为主机运行 Lan-Mouse。

·在 HTPC 上作为客户端运行 Lan-Mouse。

·躺在沙发上，用笔记本的键鼠直接控制 HTPC 播放电影、浏览网页，无需额外购置无线键鼠。





效果： 打造了一个极致便捷的家庭娱乐体验，降低了操作门槛。

模板库和社区资源： Lan-Mouse 社区虽然没有提供官方模板库，但 GitHub Discussions 中有大量的配置分享和疑难解答，鼓励用户积极探索并复用社区中的优秀实践。 如果你对 视频教程 感兴趣，可以到 Bilibili 或 YouTube 搜索 “Lan-Mouse”，有许多上手教程可以观看，大大降低了学习门槛。



实用指南

部署 Lan-Mouse 非常简单，你可以选择多种方式：

 1. 通过预编译二进制文件：


访问项目的 GitHub Release 页面，下载对应操作系统的稳定版本。

2. 通过源代码编译：

确保你的系统安装了 Rust 环境（可通过 rustup 安装）。



bash

git clone https://github.com/rust-desk/rust-desk cd rust-desk cargo build --release



3. Docker 部署 (部分社区用户贡献，非官方主流推荐，但可行)：



bash
