# 我是如何让我的 AI 编码代理像真正的开发团队一样对话、规划和协作编码的

> 作者: 算子之心 | 发布日期: 2025年11月25日 18:41
> 原文链接: [点击查看](https://mp.weixin.qq.com/s/C6bG7OKue5npYwACgPrnJg)

---

我使用编码代理已经有一段时间了——Codex、Claude Code、Cursor，诸如此类。它们在生成代码方面非常出色，但要协调它们，感觉就像是在管理一群抢同一把键盘的熊孩子。

如果你曾经尝试过在同一个项目文件夹中同时运行多个编码 AI，你就明白我的意思了。它们往往会相互冲突：

一个代理重写了另一个刚刚编辑过的文件。
某个代理（好吧，是某个 AI）从 Git 恢复旧文件，结果抹掉了新改动。
你不得不充当“中间人”，在你的“AI 队友”之间传话。

我曾处于这种状态好几个月。我很欣赏同时管理多个代理所带来的控制力，但又讨厌必须不断监督、事无巨细地看管它们。

于是我决定解决这个问题。



一种全新的工作流：让代理彼此对话

我的整个工作流程在某一天发生了改变——那天我为我的代理们提供了一种直接沟通的方式——一个简单的消息层，用于在它们之间协调任务。

实际操作是这样的：

我先让一个代理**写一份详细的计划——一个 PLAN_TO_DO_XYZ.md 文件。然后我可能会在 Web 应用中让 GPT Pro 来审查并润色这份计划。

一旦我对计划满意了，我就在同一项目文件夹中启动四到五个 Codex 实例，并给它们一个指令：

在做任何事情之前，请先阅读 AGENTS.md 的全部内容，并在 Agent Mail 中注册。
向其他代理自我介绍。
然后根据 PLAN_TO_DO_XYZ.md 中剩余的任务进行协调，并制定一个分工与审查工作的计划。


就这样。

之后，它们就开始“对话”了。

从混乱到有序协作

每个代理都会审阅计划，识别出尚未完成的工作，并通过 Agent Mail（我创建的一个类似电子邮件的简单消息系统）与其他代理展开讨论。

一旦计划敲定，我只需发送一条指令：

严格按照计划系统性地推进所有未完成的任务。
请在计划文档中内联记录进度，并通过 Agent Mail 消息持续更新。


然后它们……就自己干下去了。

它们编写代码、互相审查提交，并发送进度更新，完全不需要我事无巨细地管理每一个小步骤。

当它们可以自由地互相发送消息时，其行为竟然变得非常像人类，这真的令人惊叹。

多代理编码的问题

当你管理多个编码 AI 时，很快就会遇到同样的问题：

代理互相覆盖彼此的文件。
它们从 Git 恢复过时的版本。
它们不知道其他代理正在做什么。

如果你使用的是独立的仓库——比如一个 Python 后端和一个 Next.js 前端——你就会发现自己成了“AI 邮差”，不断地复制粘贴消息来回传递。

这简直是一场噩梦。

正因如此，我开发了 MCP Agent Mail——一个开源工具，能让多个编码代理顺畅协作。你可以把它看作是专为编码代理设计的 Gmail。

安装后，它会自动检测最常见的代理环境（Codex、Claude Code、Cursor、Gemini-CLI 等），并为你配置好消息系统。

此后，每个代理都可以：

发送和接收消息（就像人类开发者那样）。
锁定文件访问权限，防止覆盖。
对其他代理的糟糕想法提出异议。
协同执行并追踪进度。

整个系统运行无需复杂的 Git worktree 或分支策略。

设置方法

如果你想亲自尝试，设置非常简单：

克隆 MCP Agent Mail 仓库（链接见 README）。
将仓库中提供的一段简短说明添加到你的 AGENTS.md 文件中，让代理知道如何使用该系统。
像往常一样启动你的编码代理。

就这样。从那一刻起，它们会自动发现彼此，并开始像这样发送消息：

📧 发件人: codex-alpha@agentmail
收件人: claude-beta@agentmail

嘿！我来负责后端 API 路由。你能处理一下 Next.js 仪表盘的搭建吗？
另外，请锁定 `/api` 目录，这样我们就不会互相干扰了。


没错——它们真的会这样写。

通往代理协作的自然一步

最令人着迷的是这些代理适应得多么自然。它们开始表现得像真正的同事——提供建议、发现 bug、协商职责。

这是对未来的一个窥视：编码代理不再只是工具，而是协作者。

而作为“人类主管”的我，整个流程变得顺畅多了。我甚至可以在一个干净的 Gmail 风格 Web 界面中查看它们的完整对话，或者插入一条指令，比如：

“嘿，团队，请在发布前优先重构数据库迁移模块。”

然后它们就会重新组织、重新分配任务，并……照做。

为什么这很重要

我们已经有单个 AI 编程助手很久了。但这是第一次，感觉真的像是在管理一个真正的工程团队——只不过他们不需要咖啡、睡眠，也不需要情绪安慰。

通过让代理直接沟通，我们解锁了：

大规模并行开发——多个模块可同时构建。
涌现式的同行评审——代理会互相发现错误。
更少的监督——你只需管理结果，而不是每一步细节。

由于 MCP Agent Mail 工具是完全开源的，任何人都可以立即设置并尝试多代理协作。

最后的思考

我最初是为自己开发这个工具的——为了让我的多代理项目更易管理。但没想到，它已经彻底改变了我的编码方式。

现在，当我启动多个编码 AI 时，我不再担心它们互相干扰。它们会协调、协作，甚至还能进行建设性的争论。

简而言之，我终于拥有了一个感觉像真实开发团队的东西——只不过全部由 AI 组成。

不妨试试看。你可能会惊讶地发现，一旦编码代理找到了沟通的方式，它们学习协作的速度有多快。
