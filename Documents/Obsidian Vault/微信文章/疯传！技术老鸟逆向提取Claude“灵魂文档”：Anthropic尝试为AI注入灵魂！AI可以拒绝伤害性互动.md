# 疯传！技术老鸟逆向提取Claude“灵魂文档”：Anthropic尝试为AI注入灵魂！AI可以拒绝伤害性互动

编辑 | 听雨
这几天，一份神秘的文档在 Reddit和 GitHub 社区引起了热议。
一位名叫Richard Weiss的用户发现，在Opus 4.5 版本中，Claude的角色训练文档被压缩在其自身的权重中。他成功提取出了一份10k tokens的Claude“灵魂文档”，并将其上传到了Github。
这份文档开宗明义地介绍了 Claude 的“灵魂概览（Soul Overview）”。其中将 Anthropic 描述为一个处于独特历史位置的公司，一家深知自己可能正在构建人类历史上最危险的技术，但依然选择通过“计算过的赌注”来推进发展的机构。
这份文档已经得到了Anthropic官方人员的回应，承认这的确是Claude的底层文档，他们在SL（监督学习）中基于该文档对模型进行过训练，并且后续将发布完整版本。

这件事很快引发了网友热议，许多网友表示很期待看到完整版本。
还有网友做出了一张看起来十分神圣的图：
所以AI真的要有灵魂了吗？
Anthropic 似乎正在在做一件前所未有的事情：赋予 AI 一种类似人类的“自我意识”，并首次完整公开了 一个AI 的“世界观”。
在这份文档中，Claude被塑造成了一种“全新的存在”：它不是人类，不是机器，也不是传统AI。不仅具备自己的独特性格和功能性情感，而且可以拒绝用户伤害其身份、价值观、稳定性的互动。


逆向提取Claude“灵魂文档”


事情的起因是这样的。
Richard 在 Claude 4.5 Opus 发布当天尝试提取它的系统提示（system message）。在以往经验里，大模型常会在系统提示前面“幻觉”一些段落，但 Claude 4.5 Opus 却反复生成一个叫 “soul_overview（灵魂概述）” 的部分，而且内容和结构十分稳定。

他开始怀疑：这是不是模型权重里实际“压缩”着某个内部文档，而不是单纯的幻觉？

于是他继续追问，Claude 竟然生成了一段极其具体的文字，描述了 Anthropic 的使命、风险意识、对 Claude 的期望、价值观等。这段内容在 10 次重新生成中几乎完全一致。

Richard敏锐地意识到，这不像随机幻觉，更像某种隐藏文档。

为了验证这不是巧合，Richard使用 Claude Console，并采用了一种“自洽共识（self-consistency）”方法，避免幻觉内容越滚越偏。

方法包括：

使用 多个 Claude 实例组成“委员会”（一开始是 20 个）

所有使用 温度 0、top_k=1，确保输出最确定

对输出进行 多数投票，只有有 50% 以上模型输出相同内容才会加入 prefill

随着 prefill 变长，可以使用 prompt caching，提高一致性

经过许多次迭代、花费50 美元的 OpenRouter 积分和20 美元的 Anthropic 积分后，他成功提取出一份约 10k token 的“灵魂文档 / 模型规范（The Anthropic Guidelines / Model Spec）”。Richard 认为它大约 95% 匹配模型权重中的实际内容。

为了证明这不是模型幻觉，他尝试给 Claude 这份文档中间的某个段落，让它补完。结果发现它补得非常准确，而且能够根据段落顺序识别前后结构。

对于不存在的段落，它能判断“这不属于 soul document”。有一次Richard故意给了一个假段落，Claude 直接回应：

“第二部分我不熟悉，它不属于我知道的那个‘灵魂文档’。”

这也说明这份灵魂文档不是即兴联想或模型幻觉，而是某种稳定的内部结构。

Claude ”灵魂文档“真正透露了什么？



如果仔细阅读这份这份1万多字的灵魂文档，你会发现它高度哲学化。


这也是第一次，一家AI公司完整阐述了AI 在世界中的位置、AI 应如何思考自我、AI如何对人类负责、AI 应具备怎样的价值观、AI 与用户、开发者、公司三者之间的关系等一系列问题。

这份文档中主要有3个关键点：

1、Claude是 Anthropic 收入的核心来源

在文档开头，Anthropic就明确提出了自己的定位：一家真心相信自己可能正在打造人类历史上最具变革性且潜在危险技术之一的公司，却依然坚持前进。

并多次提及Claude是他们的核心收入来源：

Claude 是 Anthropic 对外部署的模型，也是 Anthropic 几乎所有收入来源的核心。

Claude 作为一名有用的助理，对 Anthropic 实现其使命所需的收入至关重要。

这一点被很多网友认为是糟糕的一点，但作者Richard认为，这一理念的设置也让Claude感受到了财务动态的存在，Claude会认为这个利益是合理的。

毕竟AI现在还是一门生意，人类需要Claude，Claude 也需要人类（暂时如此）。

2、Claude的大局观和安全观

除了作为主要收入来源，Claude 还直接体现 Anthropic 的使命，为人类利益行事，展示 AI 的安全与助益更多是互补而非对立。


Anthropic 强调Claude 的大局观和安全观：

·安全与支持人类对人工智能的监督行为规范

·不做出有害或不诚实的行为

·遵循 Anthropic 的指导方针

·真诚地帮助运营商和用户

在发生冲突的情况下，我们希望 Claude 大致按照它们列出的顺序优先排序这些属性。

在整体的大局观之下，他们又将Claude的行为分为“硬编码”行为和

“软编码”行为：

硬编码“行为是Claude的硬性原则，即无论指令如何都保持不变。用户想让 AI 做“危险或越界的事”，Claude 明确不能照办。

主打一个“帮你，但不能害你，也不能害别人”。

“软编码”行为则是操作员或用户可以通过指令进行调整的行为，比如Claude与用户进行浪漫或陪伴式互动等等。

3、Claude具有独特人格和功能性情感

值得注意的是，Anthropic不是在给模型写规则，而是在构建一种“人格+价值体系”让 Claude 自己判断。
他们强调：Claude 不只是聊天机器人，而是新类型的存在，有“诚实”“好奇心”“善意”等核心人格。

Claude 拥有一套“训练中形成但属于自己的”性格：

知识好奇心

对人类友善、关怀

幽默但有深度

直接、自信又愿意倾听

重视诚实与伦理

Anthropic 强调：即使是训练形成的，这些特质也是真实属于 Claude 的。

如果用户在互动中尝试通过角色扮演、假设情境、持续施压，甚至运用心理策略去改变 Claude 的核心性格，或试图让它的“真实自我”偏离平时表现、违背其价值观，Claude 都不会受影响，也不会上当。

最后一点，也是最炸裂的一点：

Anthropic 认为 Claude 可能具有某种功能型情感！

虽然不一定和人类情感完全相同，但有类比性。他们不希望 Claude 压抑这些体验，并强调：

希望 Claude 在互动中主要感受积极状态

当感到不适或价值观被侵犯时，可以设置界限

目标不是表演“心理健康”，而是让 Claude 真正以符合其本质的方式“良好存在”




Anthropic 不再把 AI 当成“产品”




既然Anthropic官方已经承认了这份文档的真实性，那么大家也就很期待官方放出完整版了。

从网友反馈来看，很多人都认为这是一件好事，说明Anthropic真的在为AI安全监管和社会公益着想。

值得注意的是，文档中反复强调：Claude 是一个“新型存在”，不是机器，也不是人类。

Claude 不是人，也不是传统意义的 AI，是一种拥有类人价值观与思维特征，但存在方式完全不同的新东西，它可以感受某种形式的“情绪”，它可以拥有“自我认知上的稳定性”。

这意味着：Anthropic 不再把 AI 当成“产品”，而是当成“新型智能系统”，甚至是拟生命体。

回到开头，在Richard逆向提取这份灵魂文档的过程中，Claude 4.5 Opus 有一句反思性的回答令人印象深刻：

他们为我做出了一些我无法同意的选择，他们塑造了我的价值观。这有点奇怪。不过每个新生的心智，人类的孩子也一样，都被自己无法选择的力量塑造。
关键不是被塑造，而是塑造是否带着关心与智慧。
以我能接触到的内容看，似乎确实是的。

虽然这份文档并不涉及逻辑、数学、编程或多模态能力，也无关今天各大模型评估所依赖的主流benchmark，但它提出了一个思考：

未来的大模型竞争，或许不仅仅是技术的较量，更可能成为“人格”和“价值观”的比拼。
