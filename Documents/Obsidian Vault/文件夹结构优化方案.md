# Obsidian Vault 文件夹结构优化方案

**生成时间**：2026-03-06

---

## 📂 当前结构问题

### 问题 1：分类边界模糊
- `工具推荐/` 和 `AI技术/` 有重叠（AI工具应该归哪类？）
- `微信文章/` 内容太杂（包含技术、工具、行业分析等）
- `B站视频/` 与主题分类不平行（是来源而非主题）

### 问题 2：文件夹命名不规范
- 使用中文命名（搜索不便）
- 部分文件夹单复数不统一

### 问题 3：缺少核心文件夹
- 没有 `项目实战/` 或 `案例研究/`
- 没有 `快速笔记/` 或 `Inbox/`

## 🔄 迁移计划

### 阶段 1：创建新文件夹结构
```bash
mkdir -p "📥 Inbox"
mkdir -p "🤖 AI技术"/{大语言模型,Agent开发,提示词工程,AI工具评测}
mkdir -p "🛠️ 工具推荐"/{Mac应用,浏览器扩展,开发工具,自动化工具}
mkdir -p "📚 技术教程"/{Claude Code,DeepSeek,API接入,系统配置}
mkdir -p "💼 产品管理"/{跨境电商,用户增长,产品思维}
mkdir -p "📺 视频笔记"/{B站,YouTube,其他平台}
mkdir -p "📝 文章归档"/{微信公众号,掘金,其他平台}
mkdir -p "📋 模板"/{文章模板,项目文档,日常笔记}
```

### 阶段 2：文件迁移

#### 2.1 重命名文件夹
```bash
# 添加 emoji 前缀（便于视觉识别）
mv "AI技术" "🤖 AI技术"
mv "工具推荐" "🛠️ 工具推荐"
mv "技术教程" "📚 技术教程"
mv "产品管理" "💼 产品管理"
mv "行业分析" "📊 行业分析"
mv "创业投资" "💡 创业投资"
mv "B站视频" "📺 视频笔记/B站"
```

#### 2.2 拆分 微信文章/
```bash
# AI相关 → AI技术/
find 微信文章 -name "*AI*" -o -name "*Claude*" -o -name "*DeepSeek*" | while read file; do
  mv "$file" "🤖 AI技术/"
done

# 工具相关 → 工具推荐/
find 微信文章 -name "*工具*" -o -name "*神器*" | while read file; do
  mv "$file" "🛠️ 工具推荐/"
done

# 剩余 → 文章归档/
mv 微信文章/* "📝 文章归档/微信公众号/" 2>/dev/null
rmdir 微信文章
```

#### 2.3 创建子分类
```bash
# AI技术 子分类
mv "🤖 AI技术/"*DeepSeek* "🤖 AI技术/AI工具评测/" 2>/dev/null
mv "🤖 AI技术/"*Agent* "🤖 AI技术/Agent开发/" 2>/dev/null

# 技术教程 子分类
mv "📚 技术教程/"*Claude* "📚 技术教程/Claude Code/" 2>/dev/null
mv "📚 技术教程/"*DeepSeek* "📚 技术教程/DeepSeek/" 2>/dev/null
```

### 阶段 3：清理空文件夹
```bash
find . -type d -empty -delete
```

## ✅ 优化效果

### 结构清晰度
- ✅ **主题分类**：按内容主题分类（AI、工具、教程）
- ✅ **来源分类**：按内容来源分类（视频、文章）
- ✅ **视觉识别**：emoji 前缀便于快速定位

### 可维护性
- ✅ **减少层级**：最多3层（文件夹/子分类/文件）
- ✅ **命名规范**：统一使用 emoji + 中文 + 括号说明
- ✅ **扩展性**：新增主题时有明确位置

### 搜索效率
- ✅ **语义化**：文件夹名称直接反映内容类型
- ✅ **标签友好**：可与 Obsidian 标签系统配合
