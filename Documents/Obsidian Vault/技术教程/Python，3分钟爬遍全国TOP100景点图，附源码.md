## 完整内容

 

国庆假期即将到来，这次一共攒了八天假期，但又不知道去哪儿。所以，我想，不如先看看照片？于是我打开了百度图片，一个一个地搜索，同时，我还想把这些图片保存下来，但手动保存太慢了。

于是，我决定写个小程序来帮我自动下载。一个简单的 Python 脚本，只用了一个叫 requests 的库，这个Python库其实我之前写过多个涉及数据爬取的项目，都是用这个库去实现的。

下面，我就想聊聊我是怎么把这个想法变成现实的。

1.准备工作，配置环境

开始之前，你得需要配置Python环境，也很简单，环境配置可以翻翻我之前的文章。

1. 环境：你需要一个 Python 3 的环境。
2. 工具：就一个 requests 库。如果你没有，打开终端敲一行 pip install requests 就行了。
3. 结构：我只创建了一个文件 crawl_attraction_images.py，还有一个空文件夹 attraction_images 用来放我下载的景点图片。
2.AI生成热门景点列表

这里我借助DeepSeek帮我生成国内热门景点的列表，打开DeepSeek网站，输入下面的提示词：

请帮我列举100个国庆假期值得去的国内景点，生成为列表形式，如['故宫','八达岭长城',...]

接着，DeepSeek就给我返回了我想要的景点列表，这一部分主要是想让程序实现根据我的指定列表内容来搜索图片，如果是一个一个搜索就太慢了。

3.Python爬虫

接下来就是使用Python写爬虫代码，需要注意的是，服务器会检查是谁在访问它，如果它发现你是个程序，可能会把你拒之门外。所以，我们得伪装一下，用来模拟浏览器行为。

我设置了一个 User-Agent 请求头。相当于告诉服务器：“我只是个普通的 Chrome 浏览器，不是程序，我的行为是浏览器用户行为”，这样就绕过了反爬，从而返回网站信息。

headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) ...",
    # ... 其他一些头部信息
}
resp = requests.get(url, params=params, headers=headers)

服务器返回的是 JSON 数据，里面有各种数据信息，但我们只想要图片链接，就需要对返回的数据进行解析。

我仔细看了看结构，发现图片链接通常藏在 thumbURL 或 middleURL 这两个键里。于是，我写了个循环，获取图片的二进制内容。

然后，我用 'wb' 模式把这些二进制数据写入文件。'wb' 里的 'b' 很重要，它代表二进制。这样我们就可以将热门景点的图片数据保存到文件夹中了，简单快捷。

img_data = requests.get(img_url).content
with open(filename, 'wb') as f:
    f.write(img_data)

我使用的Python编译器是VS code，代码编译完成后，只需要修改这里需要爬取的内容即可，点击运行Python代码执行程序。

运行脚本，打开 attraction_images 文件夹，整整一百张图片，从“八达岭长城”到“珠海长隆海洋王国”，应有尽有，每一张景点图片都是一个潜在的目的地景点。
