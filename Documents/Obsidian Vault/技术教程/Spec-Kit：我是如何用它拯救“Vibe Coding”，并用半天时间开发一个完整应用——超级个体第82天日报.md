什么是 Spec-Kit？它如何解决问题？

刚开始使用 AI 工具编程的朋友们，可能都遇到过这种情况：跟 AI (比如 Cursor、Claude Code) 说“帮我写个XX功能”，它就开始噼里啪啦写代码。但写出来的东西经常：

理解错了你的需求
技术选型很奇怪
代码能跑但不好维护
需要反复修改和沟通

这种漫无目的、依赖直觉的开发方式，就是“Vibe Coding”（感觉式编程）。

那么，Spec-Kit 是如何改进这个流程的呢？

它把开发过程像流水线一样，拆分成了5个明确的步骤：

第1步：定规矩 (/speckit.constitution)

"这个项目要注重代码质量、性能、测试覆盖率..." → 这相当于给 AI 立下项目“宪法”，后续所有工作都得遵守。

第2步：说需求 (/speckit.specify)

"我要做个相册管理，能按日期分组，能拖拽排序..." → 在这一步，你只说想要什么功能，完全不提技术实现。

第3步：定技术方案 (/speckit.plan)

"用 React + Node.js, 数据库用 PostgreSQL..." → AI 会基于你的需求和“宪法”，提出最合适的技术栈让你确认。

第4步：列任务清单 (/speckit.tasks)

自动把方案拆解成一个个具体、可执行的小任务。 → 就像把一个大项目拆分成每日的 TODO List。

第5步：开始干活 (/speckit.implement)

AI 按照任务清单，有条不紊地逐一实现功能。 → 告别混乱，进入有序开发。

核心优势是什么？

强制 AI 先搞清楚“要做什么”，再考虑“怎么做”。

这就像你找装修公司：

❌ 错误方式: 直接说“给我装修”，结果装成了你不喜欢的风格。
✅ 正确方式: 先明确“我要北欧风格、3个卧室”，再讨论用什么材料、哪个工人。
为什么它对 Claude Code 这类 AI 特别有效?
防止 AI "想当然" - 不会一上来就自作主张选技术栈。
减少返工 - 需求和方案都确认清楚了再动手，不用反复修改。
质量有保障 - 每一步都有清晰的文档和检查清单，过程透明。
适合团队 - 产出的规格、方案、任务等文档，团队其他人也能看懂和接手。

简单说：spec-kit 给 AI 装上了一个“产品经理+架构师”的大脑，让它不再是个只会敲代码的“码农”。

实战：用 Spec-Kit + Claude Code 开发“百日照”生成器

接下来，我将演示如何一步步使用 spec-kit 开发我的“百日照”应用。

第一步：安装和初始化

在你的项目文件夹中打开终端，运行：

# 将当前项目初始化为一个 specify 项目
uvx --from git+https://github.com/github/spec-kit.git specify init --here


接下来，根据提示选择你使用的 AI 工具（这里我们选 Claude Code），它会自动完成初始化配置。

这个命令会：

创建 spec 相关文件夹和模板文件。
配置好 specify 和 Claude Code。
设置好所有需要的斜杠命令。
第二步：启动 Claude Code

在终端输入 claude 启动。启动后你就会看到 spec-kit 提供的一系列斜杠命令：

/speckit.constitution
/speckit.specify
/speckit.plan
/speckit.tasks
/speckit.implement
第三步：遵循流水线，开始开发！
1️⃣ 建立项目原则 (可选但强烈推荐)

我使用 /speckit.constitution 命令，为我的“百日照”应用创建了开发原则，这能确保 AI 在后续每一步都牢记这些核心要求：

隐私优先架构 - 所有照片和数据必须只存储在本地，绝不上传到服务器。
AI本地化与透明 - AI生图必须在本地运行，清楚显示模型、进度、资源占用。
用户体验优于技术复杂度 - 面向非技术用户（家长），提供预设模板和智能默认值。
优雅降级与错误恢复 - 生成失败时给出清晰的错误提示和重试指导。
增量价值交付 - 功能按优先级（P1/P2/P3）独立开发，每个都能单独测试和使用。
2️⃣ 描述功能需求

接着，使用 /speckit.specify 告诉 AI 你想要什么。记住，只谈功能，不谈技术！

我要做一个“百日照”桌面应用。核心功能是：用户上传一张或多张宝宝的照片作为参考，选择一个主题模板（如宇航员、童话公主），应用能在本地调用AI模型，生成一套风格统一、保持宝宝面部特征的百日照图集。应用需要有简单的相册管理功能，所有数据和图片都保存在本地。

3️⃣ 定义技术方案

需求明确后，执行 /speckit.plan。Claude Code 会基于之前的需求和原则，提出一套完整的技术方案，包括：

确定技术栈（桌面框架、AI模型集成方案）
生成项目架构设计
创建数据模型
创建快速开始指南

这是它为我生成的方案，非常符合一个本地桌面应用的需求：

4️⃣ 生成任务清单

方案确认后，使用 /speckit.tasks 命令。Claude 会自动分析 plan 文件，生成一份详细、可执行的任务列表。

5️⃣ 开始实现！

万事俱备，现在只需执行 /speckit.implement。Claude 就会像一个经验丰富的工程师，按照任务清单逐一完成代码实现。

💡 实用技巧
每一步都可以反复修改
觉得 spec 不对？直接让 Claude 修改。
技术栈想换？重新运行 /speckit.plan 即可。
使用 /speckit.clarify 澄清需求
在 plan 之前运行，可以让 Claude 主动向你提问，把需求细节弄得更清楚，避免后面返工。
检查 Checklist
每个 spec.md 文件里都有一个 Review Checklist，可以用来检查 AI 的交付物是否满足预期。
随时中断并调整
Claude 不会一次性跑完所有任务，你可以在过程中随时介入，修正它的方向。
🎯 核心要点
不要一上来就让 Claude 写代码，先走 spec-kit 流程。
Specify 阶段不谈技术，只谈业务需求。
Plan 阶段再定技术栈，让决策有理有据。
Implement 前确保 Spec 和 Plan 都已确认无误。

这样用下来，Claude 生成的代码质量会比直接vibe coding高很多，而且文档齐全、思路清晰！
