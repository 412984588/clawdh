# Express API Patterns

**10 production-ready Express.js TypeScript REST API patterns. Stop reinventing the wheel.**

Every pattern you need to build a robust Express API — routing, error handling, validation, JWT auth, rate limiting, file upload, logging, CORS/security, database integration, and API versioning.

## What's Included

| Template | Description | Tier |
|----------|-------------|------|
| 01-basic-routing | Router, typed requests, response helpers, pagination | Starter |
| 02-error-handling | Custom errors, async wrapper, global error handler | Starter |
| 03-validation-middleware | Zod schemas, body/query/params validation | Starter |
| 04-auth-jwt | JWT sign/verify, auth middleware, refresh tokens | Starter |
| 05-rate-limiting | Fixed/sliding window limiters, per-user, presets | Starter |
| 06-file-upload | Multer, type validation, magic bytes, S3 storage | Pro |
| 07-request-logging | Structured logs, request ID, correlation, slow alerts | Pro |
| 08-cors-security | CORS, security headers, CSP, CSRF protection | Pro |
| 09-database-patterns | Repository pattern, Service layer, transactions | Pro |
| 10-api-versioning | URL versioning, header versioning, deprecation | Pro |
