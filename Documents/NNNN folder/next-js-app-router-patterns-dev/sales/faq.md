# FAQ — Next.js App Router Patterns

**Q: Which Next.js versions are supported?**
Next.js 14 and 15. The patterns use the stable App Router API available in both versions.

**Q: Do I need TypeScript?**
Yes. All templates are written in TypeScript. They won't work with plain JavaScript without modification.

**Q: Can I use these in client projects?**
Yes. The license allows unlimited personal and commercial use with a one-time purchase.

**Q: Do the templates work with Turbopack?**
Yes. The patterns are standard App Router code and work with both Webpack and Turbopack.

**Q: Are there any external dependencies?**
No. The templates use only Next.js built-ins and React. You can add your own UI library, ORM, or state management.

**Q: Do you cover Pages Router?**
No. These templates are App Router only.

**Q: What's the difference between Starter and Pro?**
Starter (templates 01–05) covers the core patterns you need in every project. Pro (templates 01–10) adds advanced patterns: SEO/metadata, parallel routes, middleware, caching, and layout composition.

**Q: Will I get updates?**
Yes. Bug fixes and compatibility updates for new Next.js versions are included at no extra cost.
