# OpenAPI Spec Templates

**10 OpenAPI 3.1 specs for real API patterns. Document and generate clients immediately.**

## The Problem

Every API needs a spec. Writing OpenAPI from scratch means remembering schemas, path syntax, components, security definitions — or copying from half-remembered examples. These templates are correct, complete, and ready to adapt.

## Starter ($19) — 4 most common patterns

- **REST CRUD**: List with pagination, create, get, update, delete. Includes reusable Error, ValidationError, PaginationMeta schemas.
- **Auth API**: Register, login, JWT refresh, logout, forgot/reset password, profile update.
- **E-commerce**: Product catalog, shopping cart, order placement and tracking.
- **File Upload**: Multipart upload, presigned URL generation, file listing and deletion.

## Pro ($39) — All 10 patterns

Everything in Starter, plus:
- **Payment API**: Stripe-compatible payment intents, subscriptions, invoice listing, billing portal.
- **Social API**: Posts, likes, comments, follows, cursor-paginated feed.
- **Search API**: Full-text search, autocomplete suggestions, faceted filter counts.
- **Webhook API**: Endpoint registration, delivery logs, test event firing.
- **Admin API**: User management, feature flags, audit logs.
- **Multi-Tenant API**: Organization creation, member roles, email invitations.

## Each spec includes

- OpenAPI 3.1.0 format
- Proper `components/schemas` with `$ref` references
- Security schemes (Bearer JWT)
- Request/response examples
- README with validation and codegen commands
