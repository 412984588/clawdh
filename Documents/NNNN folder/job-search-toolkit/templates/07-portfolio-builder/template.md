# Portfolio Builder Guide

Show your work, not just your resume. A portfolio is proof that you can do the job.

## Portfolio Structure

### 1. Hero Section
```
[Your Name]
[Your Title / What You Do]
[1 sentence: what you help people achieve]
[CTA: "View My Work" or "Let's Talk"]
```

### 2. About Section (3–5 sentences)
```
[Who you are professionally]
[Your specialty/focus area]
[What makes you different]
[What you're looking for]
```

### 3. Featured Projects (3–6 projects)
Each project follows this structure — see template below.

### 4. Skills / Tools
```
[Category]: [Skill], [Skill], [Skill]
[Category]: [Tool], [Tool], [Tool]
```

### 5. Contact
```
Email: [your email]
LinkedIn: [your URL]
GitHub/Dribbble/Behance: [your URL]
```

## Project Case Study Template

Use this for each project in your portfolio:

### [Project Name]

**Role:** [Your title/role on this project]
**Timeline:** [Duration — e.g., 3 months]
**Tools Used:** [List tools/technologies]
**Link:** [Live site, repo, or demo if applicable]

**The Problem:**
[2–3 sentences: What was the challenge? Who had the problem? Why did it matter?]

**The Process:**
[3–5 bullet points: What steps did you take? What decisions did you make and why?]

**The Solution:**
[2–3 sentences: What did you build/create/deliver?]

**The Results:**
[Quantified outcomes if possible — metrics, impact, feedback]

**Key Learnings:**
[1–2 sentences: What did you learn? What would you do differently?]

**Visuals:**
[Screenshots, mockups, before/after, or demo GIFs]

## Portfolio Project Ideas (If You Don't Have Client Work)

| Type | Project Idea | Shows |
|------|-------------|-------|
| Personal project | Build a tool that solves your own problem | Initiative, technical skill |
| Redesign | Redesign an existing app/site and explain why | Design thinking, taste |
| Case study | Write up a past project in detail | Communication, process |
| Open source | Contribute to a public project | Collaboration, code quality |
| Challenge | Complete a design/coding challenge (Daily UI, Frontend Mentor) | Consistency, skill range |
| Volunteer | Do work for a nonprofit | Real-world impact |
| Tutorial | Create a tutorial teaching something you know | Expertise, communication |

## Portfolio Quality Checklist

Before sharing your portfolio:

- [ ] 3–6 projects (quality over quantity)
- [ ] Each project has a clear problem → process → result structure
- [ ] Visuals are high quality (screenshots, mockups, no blurry images)
- [ ] Mobile responsive
- [ ] Fast loading (test with Google PageSpeed)
- [ ] No broken links
- [ ] Clear contact information
- [ ] Custom domain (yourname.com looks more professional)
- [ ] Proofread all text

## What Hiring Managers Look For

| Signal | What They're Checking |
|--------|---------------------|
| Quality of work | Can you produce professional-level output? |
| Process documentation | Do you think through problems or just jump to solutions? |
| Variety | Can you handle different types of challenges? |
| Results | Did your work actually make an impact? |
| Communication | Can you explain your work clearly? |
| Taste / attention to detail | Is the portfolio itself well-designed? |
