# 90-Day Onboarding Plan

You got the job. Now make sure your first 90 days set you up for long-term success.

## The 30-60-90 Framework

| Phase | Focus | Goal |
|-------|-------|------|
| Days 1–30 | **Learn** | Absorb everything — people, processes, culture |
| Days 31–60 | **Contribute** | Start delivering value with early wins |
| Days 61–90 | **Lead** | Own your responsibilities and propose improvements |

---

## Days 1–30: Learn (The Sponge Phase)

### Week 1: Orientation

- [ ] Complete HR onboarding (benefits, IT setup, policies)
- [ ] Set up all tools and accounts (email, Slack, calendar, dev tools)
- [ ] Meet your manager — clarify expectations and communication style
- [ ] Get the org chart — understand who does what
- [ ] Ask: "What does success look like in 30/60/90 days?"
- [ ] Schedule 1:1s with all direct team members
- [ ] Review existing documentation (wiki, handbooks, READMEs)
- [ ] Find your onboarding buddy (if assigned)

### Weeks 2–4: Deep Dive

- [ ] Shadow team members in their daily work
- [ ] Review the team's current projects and priorities
- [ ] Understand the company's product/service deeply (use it yourself)
- [ ] Learn the development/work process (sprints, reviews, deployments)
- [ ] Take notes on everything — you'll forget otherwise
- [ ] Ask "dumb" questions freely (it's expected and welcomed now)
- [ ] Schedule 1:1s with cross-functional partners

### People to Meet in Month 1

| Person | Role | Why | Scheduled? |
|--------|------|-----|-----------|
| Direct manager | | Expectations, feedback cadence | ☐ |
| Skip-level manager | | Broader context, visibility | ☐ |

| Key stakeholder 2 | | Cross-team alignment | ☐ |
| HR / People partner | | Benefits, culture, support | ☐ |
| Onboarding buddy | | Day-to-day questions | ☐ |

### Month 1 Questions to Ask

| Question | Ask Who |
|----------|---------|
| "What are the team's top 3 priorities right now?" | Manager |
| "What should I definitely NOT do?" | Manager / buddy |
| "What's the biggest challenge the team faces?" | Team members |
| "How do decisions get made here?" | Manager |
| "What does the typical workflow look like?" | Team members |
| "Who are the go-to people for [X]?" | Buddy |
| "What's the company's culture around [feedback/meetings/remote]?" | Multiple people |

## Days 31–60: Contribute (The Early Win Phase)

### Goals

- [ ] Complete your first meaningful deliverable
- [ ] Identify 1–2 quick wins (small improvements you can make)
- [ ] Start participating actively in meetings (not just observing)
- [ ] Build working relationships with cross-functional teams
- [ ] Give your manager a 30-day summary (what you learned, what you'll focus on)

### Quick Win Ideas

| Type | Example |
|------|---------|
| Fix something small | Fix a bug, update outdated docs, clean up a process |
| Automate something | Create a template, script, or shortcut the team uses |
| Improve communication | Set up a status update format, create a dashboard |
| Help a teammate | Volunteer for a task someone is overloaded on |
| Share knowledge | Present something from your past experience |

### Month 2 Check-In with Manager

Prepare for this conversation:

- "Here's what I've learned so far: _______________"
- "Here's where I've started contributing: _______________"
- "Here's what I'd like to focus on next: _______________"
- "Is there anything I should adjust? _______________"

## Ongoing Habits (Start from Day 1)

| Habit | Frequency | Why |
|-------|----------|-----|
| Keep a daily work log | Daily (5 min) | Track accomplishments for reviews |
| Send manager a weekly update | Weekly | Visibility + alignment |
| Ask for feedback | Bi-weekly | Course-correct early |
| Update your resume | Monthly | While achievements are fresh |
| Reflect on what's working | Monthly | Continuous improvement |

### Weekly Update Template (Send to Manager)

```
Subject: Weekly Update — [Date]

**This week:**
- [Accomplishment/progress 1]
- [Accomplishment/progress 2]
- [Accomplishment/progress 3]

**Next week:**
- [Priority 1]
- [Priority 2]

**Blockers / Need help with:**
- [If any]
```

## Common New Hire Mistakes to Avoid

| Mistake | Why It Hurts | What to Do Instead |
|---------|-------------|-------------------|
| Trying to change everything in week 1 | Comes across as arrogant | Observe first, suggest after month 1 |
| Not asking questions | People assume you're confused | Ask early and often — it's expected |
| Isolating yourself | Relationships matter more than skills | Have lunch with teammates, join social events |
| Overpromising | Sets unrealistic expectations | Under-promise, over-deliver |
| Ignoring company culture | Cultural fit determines success | Watch, listen, and adapt |
| Not documenting what you learn | You'll forget and ask the same questions | Keep a personal wiki/doc |
