# FAQ — Cursor Rules Pack

**What is a `.cursorrules` file?**
A `.cursorrules` file tells Cursor AI how to write code for your project. It's placed in your project root and Cursor reads it automatically for every suggestion.

**What's the difference between `.cursorrules` and `.cursor/rules/main.mdc`?**
`.cursorrules` is the legacy format used by Cursor < 0.43. `.cursor/rules/main.mdc` is the new MDC format introduced in Cursor 0.43. Both files are included in every template — use whichever your Cursor version needs.

**Do these work with Cursor's latest version?**
Yes. Both legacy and new MDC formats are included for compatibility.

**Can I customise the rules?**
Absolutely. These are starting points. Open the file and add your project-specific conventions.

**Do these work with other AI editors (Windsurf, GitHub Copilot)?**
The concepts translate, but the file format is Cursor-specific. Other editors have their own config formats.

**Is there a rule for [my stack]?**
The Pro tier covers React/TS, Next.js, Node.js, FastAPI, Go, Vue 3, SvelteKit, Monorepo, Documentation, and Code Review. If your stack isn't listed, the closest rule set is usually a good starting point to customise.
