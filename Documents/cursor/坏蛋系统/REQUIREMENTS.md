# 个人收款平台发现系统 - 需求文档

## 🎯 项目概述

本项目旨在构建一个**自动循环的智能发现系统**，持续不断地发现符合4项严格验证标准的个人收款平台。

## 🚀 核心工作流程

### 无限循环模式
**自动执行**: 发现数据源 → 找平台 → 验证 → 优化 → 下一轮 → 无限循环

### 4项验证标准（不可改变）
1. ✅ **个人注册能力**：支持个人用户注册成为收款方
2. ✅ **支付接收能力**：允许接收他人付款（打赏、购买、捐赠等）
3. ✅ **自有支付系统**：使用自有支付系统，避免外部网关依赖
4. ✅ **服务美国=ACH银行转账**：服务于美国市场，支持ACH银行转账

## 🤖 Agents团队架构

### 当前系统：3-Agent智能协作架构

#### 🔍 Scout Agent（发现者）
**职责**：基于真实数据源发现新平台
- **数据源**：Stripe Connect、PayPal Partners、Product Hunt等
- **模式**：基于已验证平台生成相似候选
- **能力**：实时爬取、去重检查、质量评估

#### 🧠 Analyzer Agent（分析员）
**职责**：深度分析平台潜力
- **算法**：基于184个已验证平台的成功模式学习
- **评分**：16维特征评估、潜力排序
- **输出**：按优先级排序的候选平台列表

#### ✅ Validator Agent（验证者）
**职责**：严格执行4项验证标准
- **验证模式**：快速预筛 → 智能预判 → 深度验证 → 二次确认
- **质量控制**：只输出4/4完全符合的平台
- **标准**：严格遵循4项验证标准，永不妥协

### 🔄 协作流程
```
Scout → Analyzer → Validator → 优化 → 下一轮
  ↓        ↓         ↓        ↓
发现平台 → 潜力分析 → 严格验证 → 策略调整
```

## 📊 性能指标

### 当前成就
- ✅ **已验证平台**：184个（截至2025-12-05）
- 📈 **发现效率**：每轮平均发现5-15个候选
- 🎯 **验证通过率**：10-25%（严格4/4标准）
- ⚡ **自动化程度**：100%无人值守

### 持续优化机制
- **自适应学习**：基于验证结果自动调整策略
- **数据源管理**：动态评估、扩展、清理数据源
- **阈值调整**：根据成功率智能微调验证标准

## 🗂️ 数据管理

### 核心文件
1. **`verified_platforms.json`** - 已验证通过的平台
2. **`rejected_platforms.json`** - 验证失败的平台
3. **`pending_platforms.json`** - 待验证平台队列
4. **`data_sources.json`** - 数据源清单和状态
5. **`validation_summary.md`** - 验证经验总结
6. **`strategy_updates.md`** - 策略变更记录

### 实时同步
- **自动保存**：每个验证完成后立即更新
- **版本控制**：Git提交记录所有变更
- **数据完整性**：去重检查、备份机制
