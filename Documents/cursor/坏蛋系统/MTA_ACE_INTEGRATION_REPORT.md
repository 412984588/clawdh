## 🧠 ACE技术架构分析

### 核心理念
ACE实现了基于论文《Agentic Context Engineering: Evolving Contexts for Self-Improving Language Models》的创新框架：

```
传统AI系统:
输入 → 模型 → 输出 (静态)

ACE系统:
输入 → Generator → 输出 → Reflector → 学习 → Curator → 更新Playbook → 下次更智能
```

### 三大智能体角色

#### 1. **Generator (生成器)**
- **职责**: 基于当前Playbook生成解决方案
- **输入**: 问题和上下文
- **输出**: 解决方案 + 推理过程
- **特色**: 引用Playbook中的相关知识

#### 2. **Reflector (反思器)**
- **职责**: 分析生成结果，识别改进点
- **输入**: Generator的输出和实际结果
- **输出**: 错误分析 + 根本原因 + 改进建议
- **特色**: 深度反思和自我批判

#### 3. **Curator (扩展器)**
- **职责**: 更新和扩展Playbook知识库
- **输入**: Reflector的分析和建议
- **输出**: Playbook的增量更新
- **特色**: 智能知识管理和积累

### Playbook知识库
- **结构**: 分层的bullet条目系统
- **内容**: helpful/harmful计数器
- **动态**: 持续演化的知识体系
- **持久化**: 跨会话的知识保留
